Заказ покупателя

Документ Заказ покупателя предназначен для регистрации заказа покупателя и резервирования заказанных товаров для последующей продажи.

Пользователь может оформлять заказы покупателей, если Администратор установил флаг Заказы покупателей на странице Продажи раздела Администрирование. Если на этой же странице Администратор установил флаг Резервирование по заказам покупателей и флаг Контролировать остатки и резерв товаров на складе в дополнительных правах пользователя, то данный пользователь может резервировать товары по заказу покупателя.

Список документов можно открыть по ссылке Продажи - Розничные продажи - Заказы покупателей.

Для удобства работы Пользователь может отфильтровать список смен по магазину, состоянию, ответственному выбрав нужные значения в соответствующих полях шапки списка. С помощью сервиса Найти, который запускается из командной панели списка, Пользователь может выбрать нужные строки, задавая поиск по значениям различных реквизитов.

На основании выбранного заказа Покупателя можно создать – Заказ на перемещение, Заказ поставщику, Ордер на перемещение товаров, Чек.

Из меню Печать Пользователь может сформировать и распечатать нерегламентированную форму Заказ покупателя.

Новый документ можно ввести с помощью кнопки Создать в командной панели списка документов.

Заполнение шапки документа

В шапке нового документа заполняются следующие поля:

· Магазин - место получения заказа;

· Склад сборки заказа - склад магазина, на котором будет храниться товар по заказу покупателя до момента продажи;

· Организация, от имени которой планируется оформить продажу;

· Дисконтная карта покупателя (если она у него есть);

· Продавец, который принял заказ. Может отличаться от ответственного, который ввел заказ в программу;

· Желаемая дата продажи - дата получения заказа покупателем, по умолчанию - текущая дата;

· Статус заказа.

При создании документа вручную ему по умолчанию назначается статус Согласован, поскольку считается, что он создается в результате непосредственного общения с покупателем. Если заказ требует дополнительного уточнения, то Пользователь может изменить статус на Не согласован. Заказ, введенный на сайте интернет-магазина и поступивший на обработку, имеет статус Не согласован. После согласования состава заказа с покупателем, пользователь изменяет статус на Согласован.

Состояние заказа отображается в виде строки рядом с полем Статус. Состояние изменяется при изменении статуса, при резервировании товаров и при проведении продаж по заказу.

Некоторые из полей шапки заполняются автоматически, если перед созданием нового заказа в списке заказов был задан отбор или если соответствующие значения установлены в настройках Пользователя.

Заполнение табличной части

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары;

· загрузить из терминала сбора данных;

· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

При подборе товаров Пользователь вводит их количество. Пользователь также может вручную изменить количество в строке табличной части. Если для отобранного товара ведется учет по характеристикам, то Пользователь при подборе выбирает соответсвующие характеристики и может вручную отредактировать их в строке табличной части.

Для отобранных товаров по умолчанию заполняются их текущие розничные цены. Пользователь может вручную изменить цены в строках или заполнить цены значениями доступных ему видов цен с помощью команды По виду цен из меню Заполнить цены командной панели табличной части.

После подбора товаров и заполнения цен в табличной части Товары Пользователь должен рассчитать автоматические скидки с помощью команды Рассчитать скидки из меню Скидки (наценки) или из командной панели табличной части Скидки. Кроме того Пользователь может назначить дополнительные управляемые или ручные скидки.

В табличной части Подарки Пользователь может подобрать подарки к данному заказу покупателя. В табличной части Дополнительно Пользователь может отредактировать параметры учета НДС и ввести текстовый адрес доставки.

Резервирование товаров

Если Администратор установил в настройках флаг Резервирование по заказам покупателей, то Пользователь может контролировать сборку заказа на складе. Для этого Пользователь нажимает кнопку Резервировать в командной панели табличной части Товары. Система вычисляет товары, для которых возможно полное резервирование (достаточно текущего остатка на складе) и проставляет в строках этих товаров флаг Резерв. Для тех строк, по котором возможно частичное резервирование система разбивает строку на две: в одной строке устанавливается возможное для резервирования количество, а в другой - остаток заказа. Пользователь может также резервировать товар вручную, проставляя флаги в строках документа. При необходимости можно разбить строчку на две, используя команду Разбить строку.

Если товара на складе сборки заказа недостаточно для выполнения заказа, то Менеджер должен организовать перемещение недостающего товара с других складов или заказать его у поставщика. В помощь Менеджеру предусмотрен отчет Состояние выполнения, который вызывается из панели навигации документа Заказ покупателя. В отчете выводится полная информация об остатках заказанного товара на складе сборки заказа, на других складах данного магазина и в других магазинах. А также отражается информация о заказанном, зарезервированном и уже проданном количестве товара. В колонке Остаток по заказу выводится количество товара, которое осталось продать по этому заказу.

В ситуации, когда товар находится в магазине, но не на складе сборки заказа, Пользователь может создать на основании заказа покупателя документ Ордер на перемещение товара. Система заполняет документ незарезервированной частью заказа.

Если товарной позиции нет в магазине, Пользователь может создать на основании заказа покупателя документы Заказ поставщику или Заказ на перемещение. Система также заполнит эти документы недостающими для выполнения заказа товарами.

После того как все товары в заказе покупателя зарезервированы состояние заказа автоматически изменяется с Ожидается резервирование на Ожидается продажа.

Продажа товаров по заказу покупателя

Если продажа товаров в магазине осуществляется через интерфейс РМК (рабочего места кассира), то для того, чтобы кассир мог автоматически заполнить чек товарами из заказа покупателя, Администратор должен в настройках РМК активировать кнопку Продажи по заказу.

Если в магазине допускается формирование чеков без помощи интерфейса РМК, то документ Чек можно создать из меню Создать на основании документа Заказ покупателя.

После того как все товары по заказу покупателя проданы, состояние заказа автоматически изменяется с Ожидается продажа на Готов к закрытию.

Отмена и закрытие заказа покупателя

Заказ покупателя может быть отменен полностью или частично. Отмену заказа товара Пользователь может оформить вручную, поставив флаг в поле Отменено выбранной товарной строки. Кроме того, Пользователь может отменить заказ для группы строк с помощью команд Отменить выделенные строки, Отменить выделенные незарезервированные строки, Отменить строки с непроданными товарами из меню Отмена строк командной панели табличной части Товары. Если Администратор установил в настройках флаг Использовать причины отмены заказов покупателей, то при оформлении отмены Пользователь выбирает причину отмены заказа из списка причин, предварительно заполненного Администратором.

Заказ, по которому все товары или проданы, или отменены, готов к закрытию. Для таких заказов пользователь может войти в документ и установить статус Закрыт, зафиксировав таким образом окончание работы с заказом.
