Справочник "Правила отбора товаров"


Справочник содержит правила формирования списка товаров для проведения выборочных инвентаризаций.

Создавая новый элемент справочника, администратор базы данных указывает схему компоновки данных из числа предоставляемых системой или создает свою. В программе доступны следующие схемы компоновки данных:

  • Обнуление остатка – в список войдут товары, у которых учетный остаток обнулился в период, начиная с указанной пользователем даты до момента формирования списка товаров. Схема имеет параметр Начало периода;

  • Номенклатурная группа – пользователь может выбрать номенклатурную группу товаров, по которой нужно провести инвентаризацию;

  • Дорогие товары – в список войдут товары, розничная цена которых на момент формирования списка выше указанной пользователем в параметре Цена товара. Пользователь может также исключить из списка товары, проверенные при других инвентаризациях за указанный период с Даты начала до момента формирования списка;

  • Списанные товары – в список товаров войдут товары, списанные по документам списания с указанной пользователем аналитикой хозяйственной операции за указанный пользователем период с даты начала периода до момента формирования списка. Так же как и в схеме по дорогим товарам, пользователь может исключить из списка товары, проверенные при других инвентаризациях за тот же период;

  • Отрицательные остатки – в список для инвентаризации войдут товары с отрицательным учетным остатком на момент формирования списка;

  • Попытки продаж, превышающих остаток – правило, аналогичное отрицательным остаткам в случае, если для магазина запрещены продажи количества, превышающего остаток (т. е. установлены флажки Контролировать остатки товаров на складе в регистре Дополнительные права пользователей для всех пользователей, оформляющих продажи, и Контролировать остатки товаров при закрытии чека в справочнике Настройки РМК при использовании режима РМК). При установленной настройке Фиксировать попытки продаж, превышающих остаток на странице Продажи раздела Настройка и администрирование информация о таких случаях сохраняется в программе, и на ее основе формируется список товаров для инвентаризации.

    Назначаемый по умолчанию параметр отбора Выводить пересчитанные товары данного приказа предназначен для того, чтобы в каждый последующий пересчет не добавлялись товары из предыдущих, то есть те, которые уже были пересчитаны ранее.

    Предварительно настроенные правила формирования списков товаров указываются в документе Приказ на пересчет товаров.
