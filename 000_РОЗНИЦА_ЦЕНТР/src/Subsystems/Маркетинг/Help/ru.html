Ведение справочника номенклатуры.

Справочник номенклатуры является центральным справочником торговой системы. В нем хранится вся информация, необходимая для заказа, оприходования и продажи товара, находящегося в ассортименте сети. Для структурирования данных и уменьшения комличества ошибок при ручном вводе информации используется ряд вспомогательных справочников, таких как:

  • Классификатор единиц измерения
  • Справочник упаковок
  • Шаблоны ценников и этикеток
  • Справочник видов номенклатуры

Виды номенклатуры - классификатор, который используется для ообобщения свойств товаров и услуг. Например, с помощью вида номенклатуры можно задать уровень детализации ведения товарных остатков - в разрезе номенклатуры или характеристик номенклатуры. Для вида номенклатуры также описываются дополнительные реквизиты и сведения, которые нужно ввести в карточку номенклатуры.

Как создать иерархию справочника номенклатуры?

Чтобы создать новую группу номенклатуры нажмите кнопку "Создать" на командной панели списка номенклатуры, введите наименование и выберите группу номенклатуры, в которую войдет создаваемая группа.

Какая информация вводится в карточку номенклатуры?

В карточку новой позиции номенклатуры вводится следующая информация.

  • Наименование. Название товара или услуги, которое используется для быстрой визуальной идентификации номенклатуры в процессе работы с программой.
  • Полное наименование. Наименование товара и услуги, которое будет печататься во всех документах.
  • Артикул. Артикул позиции номенклатуры.
  • Единица измерения. Единица измерения, в которой ведется учет складских остатков товаров и указывается количество номенклатуры в одной упаковке. Для заполнения единицы измерения используется справочник Классификатор единиц измерения.
  • Группа. Группа, в которую входит данная позиция номенклатуры в соответствии с классификацией (иерархией) товаров в справочнике Номенклатура.
  • Вид номенклатуры. Указывается вид, к которому следует отнести данную позицию номенклатуры. При выборе вида номенклатуры автоматически заполняются параметры данного вида номенклатуры: тип номенклатуры (товар или услуга), использование характеристик, использование упаковок. Изменение этих параметров невозможно.
  • Ценовая группа. Ценовая группа, в которую входит позиция номенклатуры. В разрезе ценовых групп номенклатуры устанавливаются Правила ценообразования.
  • Номенклатурная группа используется для Распределения товара по складам магазина. Поле заполняется значениями спарвочника Номенклатурные группы.
  • Страна происхождения выводится на ценник товара, размещаемый в торговом зале. Заполняется значениями классификатора Страны мира.
  • Вес нетто. Для номенклатурной позиции может быть указан вес без учета веса упаковки.
  • Весовой товар отмечается флажком.
  • Описание товара. В карточке также может быть размещено текстовое описание товара.

После ввода основных сведений о позиции номенклатуры необходимо нажать на кнопку Записать. После записи позиции номенклатуры для нее могут быть добавлены введены дополнительные данные: характеристики, упаковки, штрихкоды и т.д.

Как добавить изображение товара в карточку?

Чтобы добавить в карточку номенклатуры изображение, нужно файл изображения добавить в хранилище файлов (пункт меню Файлы раздела Органайзер). При добавлении нужно для для реквизита Папка выбрать значение типа данных Позиция номенклатуры и выбрать нужную позицию из справочника Номенклатуры.

К позиции может быть привязано несколько файлов изображения, из которых можно выбрать основное изображение в поле Картинка карточки номенклатуры.

Как ввести значения дополнительных реквизитов номенклатуры?

Список дополнительных реквизитов располагается на закладке Дополнительные реквизиты .

Для заполнения значения дополнительного реквизита пользователь нажимает на кнопку выбора в поле ввода реквизита . В зависимости от типа значения, указанного для свойства, открывается соответствующая форма: справочник значений свойств объекта, поле для указания даты, поле ввода числа, справочник клиентов и т.д. Если кнопки выбора нет, это означает, что поле предназначено для ввода текстовой информации.
Пользователь вводит значение нужного типа или выбирает нужное значение из соответствующего справочника.

Как ввести дополнительные сведения к номенклатурной позиции?

Список дополнительных сведений открывается при нажатии на соответствующую пиктограмму или выбора действия "Дополнительные сведения" из списка или из формы объекта. В списке указаны сведения, определенные в наборе свойств для вида номенклатуры либо определенные для данной номенклатурной позиции из числа общих свойств.

Как ввести характеристики для позиции номенклатуры?

Если для Вида номенклатуры данной позиции установлено ведение характеристик, общих для вида номенклатуры, то просмотреть список характеристик можно, воспользовавшись пунктом Характеристики меню карточки номенклатуры. Отображаемый список формируется в справочнике "Виды номенклатуры", но пользователь имеет возможность ввести недостающую характеристику непосредственно в карточке номенклатуры. Для этого нужно воспользоваться кнопкой "Создать" и ввести значения дополнительных реквизитов характеристики в открывшейся форме.

Аналогично создается индивидуальная характеристика номенклатурной позиции, если она принадлежит виду номенклатуры с ведением характеристик, Индивидуальных для номенклатуры.

Заполнение информации об упаковках товара.

Если товар может поставляться в магазин в различной упаковке, то для позиции номенклатуры определяется используемый Набор упаковок. Если для позиции номенклатуры используются индивидуальные упаковки, то из справочника "Наборы упаковок" выбирается предопределенный элемент "Индивидуальный набор упаковок".

Список упаковок, входящих в выбранный набор, можно просмотреть из пункта меню Упаковки панели навигации карточки номенклатуры. В случае индивидуального набора упаковки необходимые упаковки можно создать непосредственно в форме списка.

Как создается SKU код номенклатурной позиции?

SKU код представляет собой цифровой код продажи номенклатуры и ведется в разрезе упаковок и характеристик номенклатурной позиции. Для каждого сочетания упаковки и характеристики позиции необходимо создать SKU код.

Каждой номенклатуре с учетом характеристики можно сопоставить Штриховой код, по которому товар будет идентифицироваться при продаже на рабочем месте кассира. (РМК).

Отдельный штрих код можно создать и для каждой упаковки номенклатуры. Тип штрихкода упаковки может отличаться от вида штрихкода номенклатуры с определенной характеристикой. Например, для товарной позиции используется EAN13, а для упаковки EAN128.

Для того, чтобы ввести новый штрихкод перейдите к списку Штрихкоды в меню панели навигации карточки номенклатуры и нажмите кнопку "Создать".

В открывшейся форме выберите значение характеристики номенклатуры, упаковки и тип штрихкода. Переведите курсор в поле штрихкода и считайте его сканером или введите цифры вручную.

Важно. Штриховой код типа EAN128 имеет символы, разделяющие код на группы цифр. При считывании кода сканером разделительные символы опускаются. Чтобы вывести разделительные символы на печать их нужно внести в поле представление кода.

Как ввести информацию о составе комплекта?

Является ли товар комплектом, определяется только тем, заполнен ли для него список комплектующих ( пунк меню панели навигации), из которых он состоит.

Для ввода комплектующих нажмите кнопку "Создать" и выберите номенклатуру, входящую в состав. Для каждой комплектующей указывается:

  • Характеристика, если для нее установлено ведение хараткеристик;
  • Упаковка;
  • Количество упаковок;
  • Доля стоимости - весовой коэффициент учета стоимости комплектующей в составе комплекта.

Классификация и сегментация товаров.

При проведении анализа продаж выявляются товары с определенным характером продаж. Например, товары с устойчиво высокими продажами и товары с редкими или непредсказуемыми продажами. К товарам с разным характером реализации применяются разные методы пополнения товарного остатка.

Товары с постоянным устойчивым спросом нуждаются в регулярном пополнении. Заказы на такие товары могут быть созданы в полуавтоматическом режиме с помощью обработки "Анализ продажи формирование заказов поставщику".

К товарам с недостаточно высокими продажами или к новым товарам применяют методы стимулирования продаж в виде Маркетинговых акций.

Товары группируются в сегменты в соответствии с определенными пользователем правилами и используются в дальнейшем при создании скидок, как товары, на которые распространяется скидка.

Для каждого магазина можно указать также сегмент номенклатуры, на которые не будет действовать никакая скидка, например товары социального назначения, продающиеся по предельно возможно низкой цене.

Как сформировать сегмент номенклатуры?

Сегмент номенклатуры используется для задания списка номенклатуры при управлении скидками. Формирование сегмента может быть произведено тремя способами:

  • Формировать динамически
  • Периодически обновлять
  • Формировать вручную

При установленном способе формирования сегмента "Формировать динамически" -список номенклатуры не хранится в базе данных, а формируется автоматически при вызове Состава сегмента.

При способе формирования "Периодически обновлять" список номенклатуры сегмента хранится в таблицах базы данных. Состав сегмента формируется при нажатии на кнопку Сформировать или при выполнении регламентного задания Обновить сегмент номенклатуры по расписанию, определенному пользователем. Настройка регламентных заданий производится в разделе Администрирование - Настройка фоновых и регламентных заданий.

Установив "Формировать вручную", пользователь может добавить или удалить элементы состава сегмента, используя механизм отбора. При этом состав сегмента не будет обновляться при выполнении регламентного задания.

Правила формирования сегмента задаются в форме сегмента в отдельном диалоговом окне, которое открывается при нажатии на кнопку Настройка.

Как настроить правила формирования сегмента номенклатуры?

Для настройки правил формирования сегмента номенклатуры пользователь может воспользоваться предустановленной по умолчанию Основной схемой настройки. С помощью этой схемы можно произвести отбор элементов из справочника Номенклатура по группам номенклатуры, дополнительным свойствам, характеристикам и т.д.

Пользователь может также загрузить схему "По продажам и остаткам", воспользовавшись сервисом Выбрать шаблон. При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанным с продажей позиции номенклатуры: валовая прибыль, оборот, количество проданной номенклатуры, себестоимость, а также показатели оборачиваемости номенклатуры. Используя эти показатели можно например, сформировать сегмент номенклатуры по тем товарам, которые продаются плохо относительно имеющихся остатков , и назначить скидки для этих товаров.

Важно. При формировании сегментов номенклатуры можно также использовать любые произвольные запросы. Для формирования запросов используется схема компоновки данных в режиме толстого клиента.

Как определить номенклатуру, входящую в разные сегменты?

Сегменты допускают пересечения, так как один элемент может входить в несколько сегментов. Для анализа пересечений сегментов номенклатуры используется отчет Пересечения сегмента, вызываемый из меню основной формы сегмента номенклатуры.

В строках отчета выводятся элементы заданных пользователем в настройках отчета сегментов номенклатуры. Можно также указать определенные номенклатурные позиции и с помощью отчета проверить, в какие сегменты входит эта номенклатура.

Настройка правил расчета цен.

Розница поддерживает различные стратегии ценообразования для разных типов товара.

  • Стратегия назначение произвольной цены характерна для ценообразования уникального для данного товарного рынка. Маркетолог не опирается на цену закупки и может назначить цену исходя из рыночной цены аналогов или из других соображений, если аналогов нет.
  • При стратегии следования ценам поставщика стандартные цены рассчитываются по наценке на основании цен закупки данного поставщика по указанной менеджером наценке.
  • Нормированное ценообразование достигается путем расчета цен по плановой рентабельности относительно себестоимости номенклатуры.

Стратегии ценообразования обеспечиваются Видами цен номенклатуры, назначаемых для Ценовых групп номенклатуры. Для каждого вида цен назначается Правило калькуляции цены и Правила округления цены .

Настройка видов цен.

Для разграничения цен разных каналов сбыта или розничных цен магазинов используются различные Виды цен. Виды цен также используются, если нужно разграничить доступ пользователей к управлению ценами по товарным категориям.

Пример.Примеры.
1. Цены продажи мелким оптом отличаются от цен продажи в розницу в том же магазине.
В этом случае используются виды цен "Мелкооптовая" и "Розничная".
2. Розничные цены магазинов одного региона отличаются от розничных цен магазинов другого региона.
Виды цен: "Розничная цена Москва" и "Розничная цена Урал"
3. Цены на одежду и обувь формируют разные менеджеры центрального офиса.
Виды цен: "Розничная цена Обувь" и "Розничная цена Одежда".
4. На часть номенклатуры цены назначает менеджер магазина, остальные цены управляются менеджером центрального офиса.
Виды цен: "Цена центрального офиса", "Локальная цена магазина №1" и т.п.

Создание нового вида цен.

При создании новго вида цен пользователь указывает Наименование цены, которое должно достаточно полно отражать ее назначение.

Пример.Пример.
- Розничная цена Урал
- Розничная цена офиса
- Розничная цена магазина №1
- Мелкооптовая цена

Идентификатор цены необходим при описании формулы расчета цены. Идентификатор формируется автоматически при вводе наименования цены и может быть откорректирован пользователем.

  • Установленный флажок Цена включает НДС означает, что цена номенклатуры содержит НДС.
  • C помощью флажка Использовать при продаже управляется список цен для выбора вида цены в документе Реализация товара.
  • Флажок Использовать при передаче между организациями отмечает цены передачи номенклатуры между организациями предприятия.

Создание ценовых групп.

Классификатор Ценовые группы служит для создания групп, объединяющих товары с одинаковыми правилами расчета цен. При создании нового элемента справочника пользователь должен задать название ценовой группы, достаточно ясно отражающей ее содержание.

Пример:
Номенклатура с ценовой группой "Наценка 20% от цены поставщика" рассчитывается на основании цены поставщика путем умножения на коэффициент к=1.2.

Для использования ценовых групп нужно установить параметр Использовать ценовые группы в разделе Маркетинг подсистемы Администрирование.

Создание правила калькуляции.

Цену номенклатуры можно задать тремя способами:

  • Ввести вручную
  • Заполнить по данным информационной базы
  • Рассчитать по формуле

Рассчитать по формуле.
Этот способ используется в случае, если вид цен имеет определенную зависимость от другого вида цен, выраженную формулой. Формулу можно ввести в поле вручную или воспользоваться Конструктором Формул. Синтаксис формулы расчета можно проверить, нажав кнопку Проверить формулу в командной панели. Если формула составлена неверно, пользователю будет выведено сообщение об ошибке.

Ввести вручную.
Если в виде цен указан данный способ формирования, то пользователь сможет ввести цены в документ Установка цен номенклатуры вручную в соответствующую колонку табличной части.

Заполнить по данным информационной базы.
При использовании этого способа цены в документе будут заполнены значениями, хранящимися в информационной базе. Для описания схемы заполнения значений вида цен данными информационной базы пользователь может воспользоваться функционалом компоновки данных. Для этого нужно войти в систему в режиме толстого клиента и вызвать Конструктор схем компоновки данных, нажав ответсвующую кнопку.

Конструктор схемы компоновки данных.

Пользователь может разработать схему компоновки данных для расчета цен самостоятельно или загрузить готовые варианты схем из внешних файлов в формате *.mxl.

  • Нажмите на кнопку Загрузить из файла и выберите файл загрузки.
  • Пользователь может изменить данные в схеме запроса и затем сохранить исправленный текст запроса для загрузки в другую информационную базу с помощью кнопки Сохранить схему в файл.

Важно. Использование конструктора схемы компоновки данных возможно только в режиме толстого клиента.

Уточнение формулы расчета цены для ценовых групп.

По умолчанию формула задается для всех ценовых групп номенклатуры. Но пользователь может ограничить ее действие, добавляя в табличную часть ценовые группы номенклатуры из справочника "Ценовые группы" и назначая для каждой из них свою формулу расчета цены.

Конструктор формул расчета цен.

Конструктор позволяет наглядным образом описать правило расчета цены с помощью функций, арифметических и логических операторов. В качестве переменных формулы расчета используются доступные виды цен. Синтаксис формулы расчета можно проверить, нажав кнопку Проверить формулу в командной панели. Если формула составлена неверно, пользователю будет выведено сообщение об ошибке.

Округление цен.

Для каждого вида цен могут быть заданы Правила округления цены, единые для всех цен или по диапазонам в зависимости от величины цены.

Для описания правил округления для всех цен перейдите на закладку Правила округления цены и нажмите кнопку Добавить. В строке табличной части появится надпись <все цены>. В выпадающем списке колонки Точность округления выберите нужное значение.

Чтобы поддержать схему округления по диапазонам цен нужно в табличной части формы создать строки, указав нижнюю границу каждого диапазона цен. Выберите нужную точность округления для каждой строки.

Для того, чтобы получившиеся значения цен выглядели единообразно, можно также определить последнюю значащую цифру цены в колонке Психологическое округление, или воспользоваться Конструктором психологического округления.

Пользователь может установить метод округления: По арифметическим правилам или Всегда в большую сторону.

Пример.Пример.
В магазине "Три толстяка" приняты следующие правила округления цен. В диапазоне
- от 0 до 3 грн округление цены не производится.
- от 3 до 10 грн. – до 30 коп.
- от 10 до 300 грн – до 1 грн
- свыше 300 – до 30 грн
Создаем 4 диапазона цен, указывая в качестве нижней границы 3, 10 и 300. В первой строке округления нет, в следующих строках указываем точность округления 1, 10 и 100 и психологическое округление 0.3, 1 и 30 соответственно.
Проверяем результат с помощью Конструктора психологического округления.

Конструктор психологического округления.

Конструктор предоставляет возможность наглядным образом сформировать правила психологического округления, устанавливая значение значащей цифры для указанного разряда. Пользователь может проверить как работает правило, вводя произвольную цену, система отобразит цену, округленную по установленному правилу.

Как просмотреть цены номенклатуры по выбранному виду цен?

Цены, назначенные по данному виду цен, можно просмотреть в списке Цены номенклатуры в меню навигации карточки вида цен. Выводимый список номенклатуры можно ограничить, введя границы периода назначения цен. В список выводятся утстановленные цены с учетом характеристик и различных упаковок товара.

Настройка правил формирования розничных цен сети.

В системе поддерживается три вида политики управления розничных цен номенклатуры:

  • Централизованное управление – цены назначаются в центральном офисе и передаются в магазины.
  • Децентрализованное управление – цены назначаются менеджерами магазинов
  • Смешанное управление – на часть номенклатуры цены назначаются в магазинах, на остальные – в центральном офисе.

Розничные цены магазинов сети задаются с помощью Правил ценообразования.

В каждом правиле может быть назначен основной Вид цен розничных продаж и виды цен уточнения по Ценовым группам номенклатуры, продающейся только в данном магазине или регионе. Управление видами цен регулируется правами доступа конкретного менеджера по ценообразованию, чем обеспечиваются различные политики управления ценами в розничной сети.

Создание правила ценообразования.

Наименование правила ценообразования отражает область действия данного правила: конкретный магазин или группу магазинов, составленную, например, по формату магазинов или по региональной принадлежности.

Выбранный пользователем из списка основной Вид цен будет действовать для всех позиций номенклатуры.

Пользователь может вывести Правило ценообразования из использования или отсрочить начало действия вновь созданного Правила ценообразования, присвоив ему Статус «Не действует».

Пример.Пример:

Для настройки создается Вид цен «Розничные цены» и указываем его в правиле ценообразования в качестве основного вида цен. Табличная часть не заполняется.

Флажок Цена включает НДС определяет содержание НДС в действующих розничных ценах магазина. В зависимости от этого флага пользователю будет предложен для выбора список доступных видов цен, содержащих или не содержащих НДС.

Уточнение розничной цены по ценовым группам.

Если в системе включена настройка параметра "Использовать ценовые группы" в Настройках параметров Маркетинга раздела Администрирование, то становится доступна табличная часть Уточнение по ценовым группам, где пользователь может перечислить ценовые группы номенклатуры, на которые действуют другие правила ценообразования и указать для них соответствующие виды цен.

Пример 1.
Децентрализованное ценообразование. Розничные цены на все позиции номенклатуры назначаются менеджером магазина и действуют соответственно в своем магазине.
- Для каждого магазина создается вид цен «Розничная цена магазина №…» или «Розничная цена региона…».
- Для каждого магазина создается Правило ценообразования «Розничные цены магазина № …», где в качестве основного вида цен указывается соответствующий Вид цен.

Пример 2.
Смешанное ценообразование. Розничные цены на часть ассортимента назначаются менеджером магазина, остальные – управляются в центральном офисе.
- Создается по крайней мере два вида цен, один из которых доступен менеджеру офиса «Розничная цена офиса», а другой «Розничная цена магазина №…» - менеджеру магазина.
- Создается ценовая группа номенклатуры «Номенклатура магазина», в которую включается номенклатура, управляемая в магазине.
- В Правиле ценообразования «Розничные цены магазина № …» в качестве основного Вида цен номенклатуры указываем «Розничную цену офиса». В табличной части указываем Ценовую группу «Номенклатура магазина» и соответствующую ей «Розничную цену магазина №…».

Если цена товара не зависит от значения характеристики, то ее можно ввести в поле Цена группировочной строки без указания характеристики. Эта цена будет автоматически зарегистрирована для всех характеристик товаров. Режим автоматического заполнения цен характеристик в документе устанавливается флагом Заполнять цены характеристик по ценам номенклатуры в меню Параметры редактирования.
В дальнейшем можно развернуть список товаров с характеристиками и указать другую цену для характеристики товара, если она отличается от основной цены товара.

Для упаковок товара.

Пользователь может назначить цену на упаковку товара, отличную от базовой, выбрав нужное значение поля Упаковка в табличной части документа. При автоматическом расчете цена за упаковку рассчитывается с учетом коэффициента, заданного для данной упаковки.

Регистрация цен.

Для регистрации цен на товары и услуги необходимо провести документ "Установка цен номенклатуры". Фактически при проведении документа назначенные в документе цены вводятся в действие и могут быть переданы в магазины торговой сети.

Если в силу принятой технологии работы магазина необходимо отсрочить введение цен в действие, то ввод цен в действие производится менеджером магазина с помощью документа "Применение цен номенклатуры".

Анализ маркетинговых акций.

В системе реализованы ряд отчетов, позволяющих провести анализ маркетинговых акций различных типов.

  • Оценка эффективности маркетинговой акции - основной отчет сравнения показателей продаж.
  • Оценка продвижения товаров - отчет, показывающий прирост продаж определенных товаров в результате проведения акции;
  • Оценка распродажи - отчет, показывающий степень снижения товарных остатков в результате проведения распродажи;
  • Оценка продаж - отчет, показывающий эффективность акций, направленных на увеличение количества покупателей и средней суммы чека.
  • Оценка применения скидок - отчет, показывающий количество и суммы предоставленных скидок до и во время проведения выбранной пользователем акции.

Анализ акций в общем случае представляет собой сравнение ключевых показателей акции до ее проведения и во время действия акций. В случае проведения акции продвижения товара, когда целью является долговременное повышение продаж определенного наименования номенклатуры, в период анализа включается также период после завершения акции.

Чтобы провести анализ конкретной акции, пользователь выбирает отчет, соответствующий типу акции, и нужную акцию из списка, открывающегося при нажатии кнопки Заполнить по акции. При этом устанавливаются параметры отчета, соответсвующие выбранной акции.

Анализ производится в разрезе установленных в отборе магазинов – участников выбранной акции.

Период отчета по умолчанию равен удвоенному (или утроенному) периоду проведения акции. Период акции рассчитывается по расписанию акции, указанному в списке магазинов, и отображается на панели отчета: дата начала – минимальная дата по скидкам в акции по отобранным магазинам; дата окончания – максимальная дата по скидкам акции по отобранным магазинам. В отчете рассчитываются для сравнения показатели по двум или трем равным периодам: "До акции", "В течение акции" и "После акции". "Период акции"включает в себя даты начала и окончания акции, из периодов "До акции" и "После акции" эти даты соответственно исключаются.

Для расчета показателей показателей отчета весь период отчета делится на выбранные пользователем интервалы, за которые оборотные количественные и суммовые показатели агрегируются. Если в период акции (до акции) входит несколько интервалов, показатель усредняется по периоду. Например: вычисляются средненедельные продажи в период акции со сроком проведения 4 недели. Цены на товар также усредняются по подпериодам отчета.

Пользователь может изменить даты отчета, но дата начала отчета не должна превышать дату начала акции минус один интервал, а дата окончания отчета быть меньше даты окончания акции, если она определена, плюс один интервал.

Помимо вышеперечисленных отчетов, пользователь может использовать отчет, показывающий предоствленные скидки в разрезе дисконтных карт: "Скидки по дисконтным картам".

Показатели подсчитываются только по товарам, указанным в сегментах скидок для данной акции, или по всем товарам, если явного указания нет, и выводятся в три колонки: до акции, за период действия акции (во второй колонке) и изменение показателя во время акции относительно первоначального уровня в процентах.

Отчет позволяет сделать расшифровки показателей, сформированные с учетом отбора и по тем же периодам, что и выведенный на экран общий отчет:

  • Отчет Оценка применения скидок - для поля "Сумма скидки"
  • Отчет Оценка распродажи - для поля "Остатки"
  • Отчет Оценка продаж - для полей "Количество чеков" и "Средняя сумма чеков".

Отчет "Оценка продвижения товара".

Назначение отчета – анализ результативности акции, нацеленной на повышение продаж товара, товарной группы, марки и прочее. Следует принять во внимание, что если акция продвижения товара производится при прочих равных условиях, то отчет действительно покажет эффективность данной акции. В противном случае другие факторы влияния не будут учтены. То есть, если планируется акция временного снижения цены на новый, только появившийся на рынке, товар, то не следует проводить эту акцию вместе с массовой распродажей других товаров. В этом случае эффект от акции продвижения будет смазан, а отчет покажет искаженные данные.

  • Сумма с/с, грн. – средняя по подпериоду себестоимость продажи товара за интервал в гривнах.

Как настроить взаимодействие скидок?

Для скидок, действующих на одну и ту же номенклатурную позицию, пользователю необходимо установить правила совместного применения. Прейдите в список Скидки (наценки) меню Маркетинговые акции раздела Маркетинг, нажмите на кнопку Создать новую группу и выберите Вариант совместного применения для всех скидок, входящих в эту группу:

  • Вытеснение: верхняя в пределах группы скидка безусловно вытесняет все расположенные ниже;
  • Минимум: применяется скидка, которая принимает минимальное абсолютное значение суммы скидки;
  • Максимум: применяется скидка, которая принимает максимальное абсолютное значение суммы скидки;
  • Сложение : итоговая скидка – это сумма скидок.
  • Умножение: первой применяется верхняя в списке группы скидка, затем вторая скидка применяется к результату действия предыдущей скидки и т. д.

Положением скидки в группе можно управлять с помощью стрелок, расположенных в командной панели.

Важно. В корневой группе установлено правило Максимум.

Алгоритм определения итоговой скидки для позиции Чека ККМ работает так:

  • По параметрам чека из всего множества действующих скидок определяются скидки, для которых выполнились условия.
  • Начиная с периферийных веток дерева скидок по правилам совместного применения последовательно для каждой вложенной группы определяется итоговый результат действия скидок для данной строки документа.

Важно. Суммируются все скидки, применение правил условного и беусловного вытеснения скидок зависит от сопоставимости вида скидок и области предоставления скидок.

Как настроить скидку?

Чтобы настроить новую скидку выберите нужную группу скидок и нажмите кнопку "Создать" в списке Скидки меню Маркетинговые акции раздела Маркетинг.

Чтобы настроить скидку нужно указать:

  • Условия предоставления скидки - условия, при выполнении которых предоставляется автоматическая скидка и может быть предоствлена ручная скидка;
  • Вид предоставляемой скидки и ее значение - описывает преимущества, которые получает покупатель при выполнении условий скидки;
  • Область предоставления скидки - товары, на которые предоставляется скидка могут не совпадать с товарами, которые перечислены в условиях скидки.

Какие условия применения скидок поддерживаются системой?

Система поддерживает следующие условия предоставления скидок:

  • За разовый объем продаж - продажи, оформленные одним документом;
  • За накопленный объем продаж - продажи за определенный период;
  • За время продаж - продажи, совершенные в определенный интервал времени;
  • За комплект покупки - за определенные товары, купленные одновременно;
  • В День рождения - за покупку, совешенную в определенный интервалвал времени, включающий День рождения покупателя;
  • Кратность чека - определенный вид лотереи, когда скидка предоставляется по номеру чека;
  • По типу покупателя - скидка определенной группе покупателей.

Как создать скидку за разовый объем продаж?

В списке условий предоставления скидки выберите За разовый объем продаж и заполните условия расчета объема продаж. Скидка может рассчитываться по Сумме или Количеству вошедших в документ товарных позиций с ограничением Не более или Не менее введенного пользователем числа. Если указан определенный Сегмент номенклатуры, то условия будут проверяться только по товарам, входящим в сегмент.

Наименование условия формируется автоматически исходя из введенных параметров условий скидки.

Как настроить скидку за накопленный объем продаж?

В списке условий предоставления скидки выберите За накопленный объем продаж и заполните условия расчета условий. Накопления рассчитываются по дисконтной карте покупателя.

Накопления рассчитываются в виде Суммы или Количества проданного по дисконтной карте товара и могут быть ограничены - не более или не менее введенного пользователем значения. Накопления могут подсчитываться как за Весь период покупок или за период, кратный указанному значению: год, полугодие, квартал, месяц. Объем продаж может быть рассчитан С начала текущего периода времени, за Прошлый период и за Прошлый скользящий период.

Если указан определенный Сегмент номенклатуры, то условия будут проверяться только по товарам, входящим в сегмент.

Наименование условия формируется автоматически исходя из введенных параметров условий скидки.

Как настроить скидку за время продажи?

В списке условий предоставления скидки выберите За время продажи продаж и заполните Время начала и Время окончания для каждого Дня недели действия скидки.

Наименование условия формируется автоматически исходя из введенных параметров условий скидки.

Как ввести скидку на комплект покупки?

Пользователь может указать в качестве условий скидки перечень товаров, которые для получения скидки должны быть куплены одновременно. Например, при покупке садового инвентаря, 2-х лопат и грабель, секатор предоставляется в виде скидки.

Выберите значение условия скидки За комплект покупки и добавьте в список Номенклатуру, указав по необходимости Упаковку и Количество упаковок для каждого товара, входящего в комплект.

Как создать скидку в День рождения покупателя?

Выберите значение условия скидки В День рождения покупателя и введите период предоставления скидки. Если дата рождения покупателя введена в его анкетные данные, то скидка будет предоставляться автоматически в период за указанное количество Дней до Дня рождения покупателя и в течение указанного количества Дней после Дня рождения.

Если информация о дате рождения не внесена в базу данных, то скидку по предъявлении паспорта можно настроить в виде ручной скидки, предоствляемой кассиром при предъявлении паспорта. .

Как настроить скидку на каждый пятый чек?

Чтобы скидка предоставлялась по номеру чека, нужно выбрать услови предоставления скидки Кратность номера чека и указать в параметрах нужное число. Скидка будет предоставлена, если номер чека делится на указанное число без остатка. Например, на пять.

Как предоставить скидку определенной группе покупателей?

Выберите в условиях предоставления скидки условие По типу покупателя. Система предоставляет возможность сформировать список получателей скидок по различным признакам:

  • Вид дисконтной карты - скидка будет предоставлена покупателям, обладающим дисконтной картой определенного вида;
  • Дисконтная карта - скидка будет предоставлена обладателям перечисленных в списке дисконтных карт;
  • Физическое лицо - скидка будет предоставлена перечисленным в списке физическим лицам;
  • Контрагент - скидка предоставляется перечисленным контрагентам;
  • Группа получателей скидки - скидка будет предоставлена всем, входящим в перечисленные группы получателей скидки.

В название скидки автоматически добавляется тип получателя скидки. Пользователь может вручную добавить краткое описание получателей скидки данного типа, чтобы различать похожие условия в списке.

Какого вида скидки поддерживаются системой?

Может предоставляться скидка:

  • Процент - указанный процент от суммы в указанной области применения скидки;
  • Сумма - абсолютное значение скидки в указанной области применения;
  • Вид цены - изменение Вида цены, используемой в документе;
  • Подарок - часть покупки или специально выделенные товары;
  • Сообщение - при выполнении условий на экран кассира выводится сообщение.

Как настроить скидку процентом или суммой?

Нужно выбрать в поле Предоставляемая скидка значение перечисления Процент или Cумма и указать в следующем поле величину скидки.
Скидка в виде процента рассчитается от суммы документа или от суммы строк документа, указанных в Области предоставления скидки.

Как настроить перевод на другой вид цен при выполнении условий скидки?

В поле Предоставляемая скидка выберите значение Вид цены и в следующем поле выберите из справочника нужный Вид цен. При выполнении условий скидки в определенных строках документа (или в целом в документе) цена будет установлена по указанному в скидке виду цен.

Как настроить скидку в виде подарка?

Скидки в виде подарков покупателю учитываются двумя способами:

  • Подарки предоставляются из числа выбранных покупателем товаров;
  • Предоставляемые подарки перечислены в отдельном списке.

В первом случае нужно установить флаг Из корзины покупателя и в следующем поле указать то количество товаров с наименьшей ценой из документа, которые будут считаться подарками. Сумма по этип позициям будет вычтена из суммы чека и распределена по оставшимся позициям в виде скидки.

Пример.Скидка "Три диска по цене двух" настраивается следующим образом:
- Выбираем предоставляемую скидку "Подарок", устанавливаем флаг "Из корзины покупателя" и количество - 1.
- Устанавливаем условие скидки: "За разовый объем продаж": "Количество в документе не менее трех из сегмента номенклатуры "Диски"".

Во втором случае в скидке указывается перечень подарков и их количество. При выполнении условий скидки должны быть выданы все перчисленные подарки.

Пример.При покупке машины предоставляется комплект зимних шин на дисках: 4 шины и 4 диска. В список подарков в этом случае нужно внести две номенклатурные строки - шины и диска и указать в обеих количество - 4шт.

Если в табличной части подарков установлен флаг Из списка, то предоставляется подарок, описанный только одной строкой списка.

Пример.В декабре при покупке канцелярских товаров на сумму более 100грн. покупателю предлагается подарок на выбор: блокнот или карандаш.
В этом случае в список подарков также вносится две позиции номенклатуры - блокнот и карандаш, для обеих позиций указывается количество - 1 и устанавливаем флаг "Из списка". При выполнении условий скидки в режиме РМК кассиру предоставится выбор из двух позиций списка подарков.

Подарки, предоставляемые из списка, могут учитываться в бухгалтерском учете как маркетинговые материалы. В этом случае они не включаются в товарную часть чека, а списываются отдельным документом.
Если же подарки из списка учитываются как товары, то они могут быть учтены в составе проданного товара. Стоимость подарков в этом случае вычитается из суммы чека и распределяется в виде скидки по всем позициям чека. В этом случае нужно установить флаг Учитывать подарок как продажу.

Пример.Если блокноты и карандаши в предыдущем примере одновременно предоствляться как подарки и продаваться по установленной розничной цене или иметь фирменный логотип и распространяться как имиджевая реклама. В первом случае выданные подарки учитываются как продажи, во втором - как рекламные расходы.

Как настроить сообщение кассиру?

Выберите в поле Предоставляемая скидка значение Сообщение и установите время появления сообщения в интерфейсе РМК: В момент расчета скидок или После оформления чека. Введите Текст сообщения.

Пример.На предприятии применяются дисконтные карты с разным оформлением. При превышении определенного порога накопления нужно произвести замену карты.
Настроим условие скидки: при достижении накоплений 10000грн. за весь период по дисконтной карте с видом "Серебряная" и установим вид предоставляемой скидки - "Сообщение" после оформления чека: "Накопление по карте "Серебряная" превысило установленный порог. Выдайте карту "Золотая". Получив сообщение кассир произведет замену карт.

Как указать товары, на которые должна быть предоставлена скидка?

Товары, накоторые предоставляется скидка составляют Область предоставления скидки. Скидка может предоствляться либо в документе в целом, либо в строке, в которой выполнились все условия скидки, подобранные в табличной части из справочника условий предоставления скидки. Помимо этого скидка может предоставляться только на определенные товары, перечисленные в сегменте номенклатуры. В сочетании с разнообразными условиями скидки можно достичь разного результата.

Пример.При покупке двух товаров из группы "Кофеварки" 2% скидка на сумму документа. Нужно выбрать условие "Количество в документе не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" "В документе".

Пример.При покупке двух одинаковых кофеварок из группы "Кофеварки" 2% на эти кофеварки. Нужно выбрать условие "Количество в строке не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" "В строке, в которой выполнилось условие".

Пример.При покупке двух кофеварок из группы "Кофеварки" 2% на вентиляторы.
Нужно выбрать условие "Количество по документу не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" область предоставления "в документе" "На сегмент номенклатуры" "Вентиляторы".

Пример.При покупке двух кофеварок из группы "Кофеварки" 2% на вентиляторы, если их не меньше двух одного наименования.
Нужно выбрать условие "Количество в документе не менее двух по сегменту "Кофеварки"" и второе условие "Количество в строке не меньше двух по сегменту "Вентиляторы"".
Установить скидку 2 "Процента" область предоставления "В строке" "На сегмент номенклатуры" "Вентиляторы".

Как настроить скидки, которые может предоставить кассир?

Скидки могут рассчитываться автоматически или предоставляться кассиром. Для подключения возможности назначения ручных скидок, нужно установить флаг Использовать ручные скидки в Настройках параметров системы раздела Администрирование.

В скидках, которые предоставляет кассир нужно установить флаг Назначается вручную. Кассир может предоставить скидку только в том случае, если выполнились условия скидки и у него достаточно прав для предоставления ручных скидок.

Для настройки прав кассира нужно выбрать нужного пользователя или группу пользователей в списке Дополнительных прав пользователя (меню Пользователи и доступ раздела Администрирование) и установить флаг Разрешить назначение ручных скидок в группе Интерфейс РМК.

Регистрация маркетинговых акций.

Запланированные маркетологом скидки вводятся в действие документом Маркетинговая акция. Непосредственно после создания, документ мигрирует в магазины, но применяться на кассах скидки будут согласно указанному в документе расписанию.

Поскольку анализ действия скидок производится в разрезе маркетинговых акций, целесообразно вносить в документ скидки, соответствующие одной определенной цели и имеющие примерно один период действия.

Пример.Раздача листовок-скидок на 8 марта будет производится только в тех магазинах сети, которые имеют отделы парфюмерии. В этом случае в документ Маркетинговая акция будут внесены не все магазины сети и действие скидки будет ограничено одинаковым для этих магазинов периодом времени.

Пример.Распродажа зимнего сезона в северных регионах страны начинаются позже, чем в южных. В этой ситуации можно ввести две Маркетинговые акции для Северных регионов и для Южных, с разным расписанием применения скидок. Но если отсрочка начала распродажи незначительна и ею можно пренебречь при подведении итогов акции, то менеджер может создать одну Маркетинговую акцию, указав для магазинов разное расписание действий скидок.

Как ввести в действие скидки?

Как ввести в действие скидки?

Скидки вводятся в действие с помощью документа Маркетинговая аукция. При создании документа Маркетинговая акция маркетолог вносит наименование акции и период ее проведения.

В табличной части слева можно перечислить участвующие в акции магазины или установить флажок Для всех магазинов, если акция носит всеобщий характер.

В табличной части справа для каждого участвующего в акции магазина перечисляются скидки, действующие в рамках данной акции и период их действия, по умолчанию совпадающий с периодом акции. Если расписание действий скидок универсальное, то устанавливается флажок Для всех магазинов одно раписание скидок. Пользователь может изменить время действия скидок в пределах периода проведения всей акции.

На закладке Описание можно ввести подробное описание акции и действий персонала магазина при ее проведении.

Пример.Ступенчатые распродажи.
Предприятие проводит распродажу зимней одежды в два этапа.
1-й этап. Распродаже со скидкой 30 % подлежат товары групп «Детская верхняя одежда», «Подростковая верхняя одежда» зимнего ассортимента с оборачиваемостью менее 60 % за период с 01.10.2008 по 25.01.2009. Дата начала действия скидки – 04.02, окончания – 03.03.
2-й этап. Распродаже со скидкой 50 % подлежат товары групп «Детская верхняя одежда», «Подростковая верхняя одежда», «Головные уборы» зимнего ассортимента, «Варежки и перчатки» с оборачиваемостью в целом по сети менее 80 % за период с 01.10.2008 по 25.02.2009. Дата начала действия скидки – 04.03, окончания – 31.03.
Перед началом каждого этапа распродажи менеджер формирует сегмент номенклатуры "Товары на распродажу" с помощью Шаблона формирования сегмента "По продажам и остаткам" и формирует для каждого этапа распродажи скидку - 30% и 50%. Поскольку распродажа производится по всей сети одновременно, то в документе Маркетинговая акция устанавливаются флаги "Для всех магазинов" и "Для всех магазинов одно расписание скидок" и для каждой скидки устанавливается период действия.

Важно. Списки товаров могут формироваться как в целом для всей сети, так и с учетом особенности реализации номенклатуры в каждом магазине. В последнем случае пользователю нужно сформировать отдельный список товаров для распродажи для каждого этапа в каждом магазине, используя параметр Магазин в шаблоне формирования сегмента "По продажам и остаткам"; на каждый этап распродажи для каждого магазина настроить свою скидку, указав сформированный для магазина/этапа сегмент и для каждого магазина настроить расписание действия своих скидок.

Важно. С помощью расписания можно приостановить действие скидки на определенное время. Например, в табличную часть скидок можно внести дублирующую запись скидки "В утренние часы" и установить сроки действияэтой скидки с 01.02 по 15.02 для первой записи и с 01.03 - для второй записи. В этом случае действие скидки будет приостановлено на период с 16.02 по 28.02.

Подарочные сертификаты.

Подарочные сертификаты – это универсальный подарок, получивший широкое распространение в таких отраслях, как розничная торговля ювелирными изделиями, косметикой и парфюмерией, бытовой техникой. Подарочные сертификаты выпускаются как в виде карт с магнитным, штриховым кодом или просто номером, так и в виде нумерованных купонов, распечатанных на бумаге типографским способом или на принтере.

На сертификате обычно указывается:

  • Номинал сертификата, который остается неизменным в течение всего срока действия сертификата.
  • Условия погашения сертификата и срок действия, который может быть ограничен определенной датой, периодом с даты продажи или не ограничен (практически не используется).
  • Серийный номер сертификата.
  • Товарный знак компании или группы компаний, в которых действует подарочный сертификат.

Продажа сертификата регистрируется на кассе ККМ только по номиналу, на них не распространяются скидки. Проданные сертификаты принимаются к оплате последующей покупки также в размере номинала.

Как создать подарочный сертификат?

Для регистрации изготовленных сертификатов нужно создать Вид номенклатуры с типом Подарочный сертификат и ввести новый элемент справочника Номенклатуры с этим видом.

Пользователь вводит наименование Подарочного сертификата, указывает единицу измерения и ставку НДС 20%. Перейдя на закладку Сертификаты, вводит номинал и срок действия сертификата. В системе поддерживаются бессрочный сертификаты, сертификаты, заканчивающие свое действие на определенную дату и действующие в течение определенного количества дней после продажи.

Перейдя на закладку Штрихкоды, пользователь может создать список штрихкодов для данного сертификата определенного типа.

Выпущенные сертификаты могут иметь индивидуальные серийные номера. Чтобы ввести в систему серийные номера нужно на закладке Сертификаты установить флажок Использовать серийные номера и указать тип используемого кода: магнитный, штриховой, смешанный. При этом станет доступна табличная часть Серийный номер, в которую пользователь может добавить записи вручную или воспользоваться обработкой Заполнить серийные номера.

Как работать с обработкой "Заполнить серийные номера"?

Для массового создания серийных номеров подарочных сертификатов удобно воспользоваться обработкой, которая создает серийные номера в диапазоне, указанном пользователем.

Магнитные серийные номера могут иметь символьные или цифровые префиксы и постфиксы, а также есть возможность указать длину цифрового кода и первый код диапазона. Последний код диапазона вычислится автоматически и отобразиться на форме.

Обработка ввода серийных номеров позволяет проверить, свободен ли намеченный для сертификатов диапазон штриховых и магнитных кодов. Если в указанных пользователем диапазонах есть коды, которые используются для товаров, дисконтных или информационных карт, будет выведен список этих кодов. Двойным щелчком на строке списка занятых кодов можно перейти к данной позиции и, если это необходимо, внести исправления.

При нажатии кнопки Создать в табличной части "Серийный номер" будет создан список номеров указанного диапазона, за исключением занятых другими объектами базы данных.

Как вести учет сертификатов?

Подарочные сертификаты могут быть оприходованы и перемещены со склада на склад и списаны по окончании срока действия.

Для правильного отражения сертификатов на складских остатках (например, в отчетах Оценка распродажи или Стоимостная оценка склада), необходимо установить розничную цену сертификата, равную его номиналу.

Как добавить серийный номер сертификата в документ?

Если сертификаты имеют серийные номера, в документе можно указать диапазон серийных номеров воспользовавшись командой Ввести подарочные сертификаты в меню табличной части документа. В открывшейся форме доступен выбор Диапазона из числа существующих или предоставляется возможность создать новый диапазон серийных номеров подарочных сертификатов.

Как регистрируется продажа подарочного сертификата?

Кассир продает подарочный сертификат как обычный товар с той разницей, что подарочный сертификат может идентифицироваться разными способами: считыванием штрихкода или магнитного кода или вводом/выбором символьного кода. После выбора подарочного сертификата появится окно выбора серийного номера с возможностью ручного ввода.

Важно. Особенность продажи подарочного сертификата – система определяет цену за сертификат по его номиналу, и ее нельзя поменять в документе.

На подарочный сертификат нельзя применить ни ручную, ни автоматическую скидку. Он не может быть в составе набора покупки для определения скидки-подарка и не может быть выдан как подарок.

Система помечает подарочный сертификат с заданным серийным номером как проданный, и второй раз его продать нельзя.

Как ввести серийный номер сертификата при оформлении продаж в режиме РМК?

При продаже сертификата или оплате покупки с помощью сертификата, кассир считывается штриховой или магнитный код. Соответствующий серийный номер сертификата будет учтен в документе автоматически.

Если серийный номер сертификата представлен в виде последовательности печатных символов, например в случае, если сертификат распечатывается на бланках, то при выборе сертификата из справочника при продаже или выборе вида оплаты "Сертификат" открывается форма ручного ввода серийного номера сертификата.

Как производится оплата Подарочным сертификатом?

При оплате покупки кассир выбирает платежное средство Подарочный сертификат и считывает код переданного ему покупателем сертификата. Система проверяет, продавался ли сертификат с этим номером и не закончился ли срок действия сертификата.

Если сертификат меньше суммы продажи, то покупатель доплачивает разницу обычным путем (наличные или банковская карта). На одну покупку может быть предъявлено несколько сертификатов. Номиналы сертификатов при этом суммируются.

Суммы продажи сертификатов учитываются как авансовый платеж покупателя в пользу торгового предприятия. Поэтому при превышении номинала предъявленного к погашению сертификата над суммой выбранных товаров остаток суммы покупателю не возвращается. Это условие должно быть описано в правилах погашения сертификатов и доведено до сведения покупателя при продаже. При этом система информирует кассира о сумме превышения номинала сертификата над суммой покупки.

По окончании смены кассир прикладывает к отчету полученные в оплату товара сертификаты, как обоснование снижения наличной выручки. Сертификат вторично не используется и хранится вместе с другими кассовыми документами.
