Настройка и обновление общих класификаторов.

Общие классификаторы - это вспомогательные справочники, которые помогают пользователю ускорить процесс ввода данных и избежать ошибок ввода текстов. Сиcтема поддерживает следующие классификаторы:

  • Классификатор "Страны мира"
  • Справочник "Виды контактной информации"
  • Справочник "Банки"
  • Классификатор "Документы, удостоверяющие личность"

Как заполнить справочник "Страны мира"?

Справочник Страны мира содержит список стран, используемый для ведения адресной информации. При первом запуске программы информация в справочнике заполняется списком стран мира в соответствии с Приложением к приказу "Держмитслужби України" от 30 декабря 2003г. № 930. Справочник используется для введения адресной информации в справочниках Физические лица, Контрагенты, Организации, Магазины, Пользователи.

Как создать новую запись в классификаторе "Страны мира"?

При создании дополнительной записи в классификаторе нужно указать трехзначный цифровой код страны и два буквенных альфа-кода, принятые в международной классификации, краткое и полное наименование страны.

Зачем нужны виды контактной информации?

Справочник служит для классификации контактной информации в справочниках системы - например, контрагентов, сотрудников, покупателей и т.д.

В списке справочника виды контактной информации, относящиеся к одному справочнику объединены в группу. В системе поддерживается контактная информация для справочников:

  • Организации
  • Пользователи
  • Физические лица
  • Контрагенты

Для каждой группы определен список предопределенных видов контактной информации, которые отмечены в списке определенным знаком. При необходимости в любую из групп пользователь может добавить необходимый вид дополнительной контактной информации. После добавления новый вид контактной информации автоматически появится в форме соответствующего справочника.

Как настроить виды контактной информации для конкретного справочника?

Для каждого вида контактной информации в поле Группа определен справочник системы, для которого предназначен данный вид контактной информации и тип контактной информации, который выбирается из фиксированного списка:

  • Адрес
  • Телефон
  • Адрес электронной почты
  • Веб страница
  • Факс
  • Другое — дополнительная информация в виде текста, например, о том, как добраться до организации.

Для полей с типом Адрес, Телефон или Факс можно установить флажок Редактировать только в диалоге. Настройка позволит ограничить ввод нового и редактирование существующего адреса формой-шаблоном ввода адреса.

Реквизит Адрес только украинский позволит ограничить ввод страны адреса Украиной.

Где хранится информация о банках?

В справочнике Банки хранятся сведения о банках, с которыми работает торговое предприятие: наименование банка, МФО, данные о местонахождении, телефоны для связи с сотрудниками, почтовый адрес банка.

Сведения о банках используются при заполнении информации в справочнике Банковские счета организации при вводе информации о банковских счетах организации торгового предприятия.

Для удобства работы пользователей и группировки записей по какому-либо признаку, территориальному или группировки филиалов одного банка, могут быть созданы группы банков.

Карточка Банка может быть создана вручную с помощью кнопки Создать или заполнена автоматически с использованием "Классификатора банков".

Как создать новую запись о банке?

При создании нового банка пользователь указывает реквизиты, необходимые при работе с банком:

  • Наименование банка,
  • МФО,
  • Данные о местонахождении,
  • Телефоны для связи с сотрудниками,
  • Почтовый адрес банка.

Как добавить банки из классификатора?

В списке банков нажмите на кнопку командной панели Добавить из классификатора.

В появившемся диалоговом окне укажите банки, которые хотите добавить в список. Нажмите кнопку ОК.

Как дополнить поддерживаемые системой перечень видов документов?

Справочник содержит список основных видов документов, удостоверяющих личность (паспорт гражданина Укранины, водительское удостоверение и т.д.) и может быть дополнен пользователем. Информация о виде документа используется при заполнении данных в удостоверении личности, предоставленном физическим лицом.

Как используется справочник номенклатурных групп?

Настройка видов номенклатуры.

Пользователь может добавить в справочник неограниченное количество видов номенклатуры, однако все они могут быть только трех типов: Товар, Услуга, Подарочный сертификат.

  • Товар. Товаром может являться не только купленная номенклатурная позиция, но и те номенклатурные позиции, которые получаются в процессе предварительной доработки (комплектации, разукомплектации). Такие позиции в дальнейшем будем называть комплектами.
  • Услуги. По услугам не ведется складской учет. Информация об услугах заполняется в табличной части документа вместе с товарами.
  • Подарочные сертификаты. Подарочные сертификаты - это особый вид бонуса, предоставляемый торговым предприятием розничным покупателям. Они имеют определенный период действия и номинал (стоимость) и могут быть проданы покупателю как обычный товар. Проданные непросроченные сертификаты могут приниматься в оплату новой покупки.

Характеристики номенклатуры предназначены для более детального ведения складского учета товаров и услуг. Для номенклатуры может быть определено использование характеристик:

  • Не использовать - Учет по характеристикам не ведется.
  • Использовать общие для вида номенклатуры - Характеристики подчинены данному виду номенклатуры.
  • Использовать индивидуальные для номенклатуры - Характеристики подчинены номенклатурным позициям.

После выбора типа номенклатуры и использование характеристик, следует записать элемент справочника. При этом автоматически сформируется запись о наборе свойств данного вида номенклатуры. Если установлено использование характеристик, набор свойств создастся и для характеристик вида номенклатуры. Щелчком правой клавиши мыши на значке просмотра можно открыть форму настройки наборов.

Для каждого вида номенклатуры может быть определен определенные шаблоны ценника и этикетки.

Как настроить наборы дополнительных реквизитов номенклатуры?

Наборами свойств номенклатуры могут быть определены дополнительные разрезы, по которым не ведется складской учет, но могут быть использованы в отчетах. Пользователь может также установить наборы свойств характеристик, используемых для ведения номенклатуры.

Наборы свойств для вида номенклатуры и для характеристик вида номенклатуры настраиваются с помощью механизма дополнительных свойств и сведений.

В открывшейся форме с помощью кнопки Подбор отберите в набор из списка дополнительных реквизитов и дополнительных сведений необходимые для данного вида номенклатуры.

Как создать список характеристик, общих для вида номенклатуры?

Если для вида номенклатуры назначено ведение характеристик, общих для вида номенклатуры, то после записи вида номенклатуры становится доступна закладка Характеристики. Прейдите на эту закладку и нажмите кнопку "Создать".

В открывшейся форме выберите значение для установленных для данного вида номенклатуры дополнительных реквизитов характеристики номенклатуры.

Наименование характеристики сформируется автоматически. Выберите один из предложенных вариантов заполнения поля.

Настройка единиц измерения номенклатуры.

В справочник «Базовые единицы измерения» вводится список используемых в магазине основных единиц измерения (штуки, килограммы, метры и т.п.).

Справочник имеет одноуровневую структуру. Новый элемент можно создать вручную или воспользоваться классификатором единиц измерения.
Для добавления данных из классификатора необходимо нажать на кнопку Подобрать из классификатора.

Откроется таблица лассификатора единиц измерения. Уже используемые единицы измерения в классификаторе подсвечены серым. Пользователь может отметить нужные единицы измерения флажком, и при нажатии кнопки ОК они будут добавлены в список единиц измерения.

Настройка набора упаковок.

После ввода информации о названии набора упаковки и единицы измерения, в которой будет измеряться упаковка, необходимо записать элемент и перейти к заполнению списка упаковок, входящих в набор (Перейти к упаковкам).

В справочнике Упаковки заполняется список упаковок для выбранного набора упаковок . Для каждой упаковки указывается количество в единицах измерения, указанных для набора упаковки, вес и размер упаковки.

Пример. Предприятие торгует конфетами (в коробках) и молочными продуктами (йогуртами). Конфеты могут поступать в упаковках по 10 коробок или в ящиках по 25 коробок. Йогурты могут поступать в упаковках по 24 штуки или в ящиках по 48 штук. В справочнике "Наборы упаковок" следует ввести два набора упаковок "Конфеты (в коробках)" и "Йогурты". Для конфет единицей измерения будет коробка, а для йогуртов - штуки.
Для каждого из наборов нужно ввести по две упаковки с соответствующим количеством единиц товара: 10 и 25 - для конфет и 24 и 48 - для йогуртов.

Настройка шаблонов ценников и этикеток.

Новый шаблон ценника или этикетки может быть создан заново или введен на основании шаблона ценники или этикетки, используемого в системе по умолчанию.

  • С помощью кнопки Создать в списке шаблонов откройте форму нового шаблона и нажмите кнопку Шаблон по умолчанию. Выберите нужный шаблон по умолчанию из списка, при этом будет загружен предустановленный в системе шаблон ценника (этикетки).
  • Укажите наименование нового шаблона.
  • В поле "Количество по вертикали" укажите то количество ценников (этикеток), которое должно располагаться на листе печати по вертикали.
  • В поле "Количество по горизонтали" укажите то количество ценников (этикеток), которое должно располагаться на листе печати по горизонтали.
  • Отредактируйте шаблон, добавив в него любое поле из списка доступных полей. При создании или редактировании шаблона используются те стандартные правила работы, которые применяются при работе с табличными документами в 1С:Предприятии.
  • Завершите создание шаблона, нажав на кнопку "Записать и закрыть".

Важно. Важно. Для каждого шаблона можно настроить свои параметры печати: принтер, размер бумаги, поля отступа и т.д.

Как используется справочник номенклатурных групп?

Справочник номенклатурных групп используется для группировки товаров по местам продажи. Эти сведения используются при автоматическом распределении товаров по торговым залам или отделам магазина.

Каждой номенклатурной позиции может быть сопоставлена определенная номенклатурная группа, которая указывается в карточке номенклатуры.

Ведение информации об организациях

Торговое предприятие может состоять из нескольких организаций, от имени которых будут оформляться документы. В программе ведется учет в разрезе организаций торгового предприятия. Для каждой организации определяется список банковских счетов и система налогообложения: общая или упрощенная.

Создание новой записи об организации.

При вводе новой организации следует указать следующие данные:

  • Наименование - краткое наименование организации, по которому будет удобно находить организацию в списках.
  • Сокращенное, Полное - сокращенное и полное официальное наименование, которое будет использоваться при печати документов.
  • Юр/Физ.лицо - указывается кем является организация с юридической точки зрения: юридическим или физическим лицом, резидентом или нерезидентом.
  • Префикс - используется для разделения нумерации документов по разным организациям В начале номера документа добавляется указанный префикс.
  • Код по ЕДРПОУ, ИНН и Номер свидетельства плательщика НДС - информация о кодах организации в соответствии с ее учредительными документами.

На закладке Адреса и телефоны вводится информация о контактной информации организации (адреса, телефоны, факс).

Важно. По умолчанию для организации настроены определенне виды контактной информации: Email, Почтовый адрес, Другая информация, Телефон, Факс, Фактический адрес, Юридический адрес. При необходимости список видов контактной информации можно расширить.

На закладке Ответственные лица вносятся фамилия и инициалы руководителя, бухгалтера и кассира организации для выведении на документы при печати.

Как внести информацию о банковском счете организации?

Выберите пункт Банковские счета организации в меню карточки организации и нажмите пиктограмму Создать.

В открывшейся форме введите номер счета и МФО банка. Корсчет, Наименование банка и Город, в котором расположен филиал банка определится автоматически по данным классификатора Банки. Пользователь может выбрать наименование для созданного банковского счета организации из числа автоматически созданных системой или ввести свое.

Как изменить систему налогообложения организации?

Нужно перейти в список Системы налогообложения в меню карточки нужной организации и создать новую запись. При этом дата по умолчанию заполнится первым днем текущего месяца, но пользователь может ее изменить. Нужно выбрать систему налогообложения (общая или упрощенная), которая начнет действовать с указанной даты.

Где можно просмотреть список операционных касс организации?

Список всех операционных касс можно открыть из пункта меню Кассы карточки организации раздела НСИ. Пользователь может применить быстрый отбор по магазину.

Как ввести информацию о главной (операционной) кассе магазина?

Чтобы создать новую операционную кассу нужно перейти к списку касс в карточке нужной организации и нажать кнопку "Создать". Пользователь может использовать быстрый отбор по магазинам - в списке отразятся все кассы, созданные для данного магазина и организации.

В открывшейся форме нужно ввести Магазин, к которому относится касса и Наименование кассы. Наименование может быть сформировано автоматически или введено пользователем вручную.

Где можно просмотреть список касс ККМ для данной организации?

Список всех касс ККМ данной организации можно открыть из пункта меню Кассы ККМ карточки организации раздела НСИ. Пользователь может применить быстрый отбор по магазину.

Как создать кассы ККМ для магазина сети?

Чтобы создать новую операционную кассу нужно перейти к списку Касс ККМ в карточке нужной организации и нажать кнопку "Создать". Пользователь может использовать быстрый отбор по магазинам - в списке отразятся все кассы, созданные для данного магазина и организации.

В открывшейся форме нужно ввести Магазин, к которому относится касса и Наименование кассы. Наименование может быть сформировано автоматически или введено пользователем вручную.

В карточке Касс ККМ также указывается рабочее место (компьютер), на котором будет работать кассир и реквизиты (регистрационный номер, серийный номер и ширина используемой ленты) установленного фискального регистратора или принтера чеков.

Важно. При установленном флаге Ручной режим формирования чеков нефискальные чеки печатаются не автоматически, а только при нажатии клавиши Печать чека в интерфейсе кассира (РМК).

Введение информации о магазинах.

Информация о магазинах сети вносится в справочник Магазины. Магазином считается как правило территориально удаленная организационная единица предприятия, предназначенная для продажи товаров розничному покупателю. Товар также может отпускаться со складов магазина мелким оптом.

В справочнике описываются настройки для работы складов магазина: склады поступления и продажи, настройки ордерной сехмы работы складов, настройки применения цен и скидок при розничных и мелкооптовых продажах.
Пользователь имеет возможность также настроить автоматическое распределение товаров по складам при поступлении в магазин и распределение товаров по складам продаж (отделам торгового зала) для автоматического списания остатков при продаже через РМК.

Пример.В магазине Обувь имеется несколько торговых залов (складов): Женская обувь, Мужская обувь, Спортивная обувь. В каждом торговом зале продается соответсвующая номенклатурная группа товаров. Для каждой номенклатуры указывается ее принадлежность к номенклатурной группе. При поступлении товара можно автоматически оприходовать его в соответствующие торговые залы на основании настроенного распределения поступлений.

При работе с конфигурацией "1С:Управление торговлей для Украины", редакция 3 в качестве управляющей системы распределительные склады, снабжающие магазины товарами создаются в Управлении торговлей и передаются в конфигурацию "1С:Розница для Украины" в качестве элемента справочника "Магазины" с типом объекта розничной сети "Склад управляющей системы". Создание складов управляющей системы непосредственно в конфигурации "1С:Розница" не предусмотрено.

Какие настройки цен можно задать для магазина?

Для каждого магазина нужно указать Правило ценообразования, согласно которому на продаваемую номенклатуру будет назначена розничная цена.

При продаже со скидкой рассчитанная цена может оказаться слишком низкой. Чтобы этого избежато можно назначить Вид минимальных цен продажи. При этом рассчитанная цена со скидкой автоматически заместиться значением минимальных разрешенных цен.

Для каждого магазина может быть создан список номенклатуры, на которую не действуют никакие скидки (Сегмент исключаемой из скидок номенклатуры). Например, это социально значимые товары, продающиеся со сниженной наценкой. Вне зависимости от условий предоставления скидки эти товары будут продаваться по назначенной розничной цене.

Чтобы избежать повышенного расхода разменной монеты, можно установить один из пречисленных в списке Порядок округления суммы чека в пользу покупателя. Например, при округлении до гривны, сумма чека, рассчитанная как сумма по товарным строкам чека с учетом скидок, будет округлена вниз до ближайшего целого значение гривен. При этом "лишние копейки" будут вычтены с позиции с максимальной суммой.

Спомощью флага Использовать применение цен можно управлять возможностью отложить применение на кассах ККМ назначенных централизованно розничных цен. При этом должна быть установлена одноименная функиональная опция в разделе Маркетинг настройки праметров системы.

Как создать склады магазина?

В карточке магазина перейдите в список складов магазина и нажмите кнопку "Создать".

В открывшейся форме введите наименование и выберите тип склада: Складское помещение или Торговый зал.

Заполните Организацию, от имени которой по умолчанию будут оформляться документы. Для склада с типом "Торговый зал" этот реквизит обязателен. В соответствии с этим реквизитом и распределением продаж по складам магазина производится автоматическое формирование чеков продажи при использовании РМК.

Как указать автоматически подставляемые значения склада поступления и склада продажи?

После того как для магазина указаны наименование и правило розничного ценообразования вновь созданную карточку магазина нужно сохранить и перейти к созданию складов магазина.

Пользователь может указать один из созданных складов для использования в качестве Склада послупления по умолчанию. Например, в магазине "Одежда" все товары проходят предпродажную подготовку, поэтому приходуются на специальный склад Предпродажная подготовка. Этот же склад используется для заполнения поля Склад документов, полученных из конфигурации "Управление торговлей для Украины".

Со склада, указанного как Склад продажи автоматически списываются остатки товаров при продаже через РМК, если не настроено распределение товаров по складам для продаж.

Как настроить использование ордерной схемы для складов магазина?

Одерная схема в магазинах используется, если фактические операции с товаром и оформление документов производится в различные моменты времени разными пользователями.

Использование ордерной схемы можно настроить для приходных и расходных операций, установив соответствующий флажок Использовать ордерную схему при поступлении или Использовать ордерную схему при отгрузке. Настройка будет действовать для всех складов магазина.

Перемещение товаров из магазина в магазин всегда производится по ордерной схеме, поэтому флаг параметра установлен по умолчанию и не дотупен для изменения.

Как указать график работы магазина?

Чтобы указать график работы магазина на следующий год перейдите к списку графиков, воспользовавшись пунктом меню Графики работы магазина в карточке магазина и нажмите кнопку "Создать".

В открывшейся форме можно выбрать Вид графика - пятидневка, шестидневка или полный и скорректировать тип дня. Для изменения типа дня двойным щелчком на нужной дате вызовите контекстное меню и выберите одно из значений: Праздник, Предпраздничный , Рабочий, Суббота, Воскресенье.

График работы магазина можно распечатать.

Что такое интервал работы магазина?

В интервале работы магазина описывается режим рабочего дня магазина: начало и окончание работы сотрудников магазина, количество сотрудников и состав работ, которые необходимо выполнить в этот день.

В системе имеется возможность настроить несколько интервалов работы магазина для будничных, праздничных или предпраздничных дней, а также любой нестандартный режим работы.

Как настроить автоматическое распределение поступления по складам магазина?

Если в магазине указан Склад поступления, то все товары по умолчанию будут приходоваться на этот склад. С помощью схемы распределения товаров по складам можно задать уточнение распределения по некоторым номенклатурным группам.

Для настройки схемы распределения товаров по складам магазина воспользуйтесь пунктом Распределение поступления меню карточки магазина.

В строках таблицы распределения показываются номенклатурные группы товаров, а в столбцах - все склады магазинов, кроме указанного в качестве основного склада.
В соответствующих графах с помощью нажатия правой клавиши мыши или клавишей ENTER указывается на какой склад должен поступать товар номенклатурной группы.

Как настроить распределение продаж товаров по складам?

Указанный в карточке магазина Склад продажи подставляется по умолчанию в документы Реализация товаров и Чек ККМ. С помощью Распределения продаж можно уточнить склады продажи для отдельных номенклатурных групп.

Для настройки схемы распределения продажи товаров по складам магазина воспользуйтесь пунктом Распределение продажи меню карточки магазина.

Таблицу распределения продаж можно Заполнить по распределению поступлений, вопользовавшись сервисом командного меню, и затем изменить нужные значения снимая или проставляя галочки в нужных графах нажатием правой клавиши мыши.

Можно также назначить рабочее место для оформления продаж товаров определенной номенклатурной группы с определенного склада. В этом случае остатки товара по продажам с этого рабочего места будут списаны с указанного склада, в то время как на любом другом рабочем месте продажи будут производится с общего склада продажи или со склада указанного для номенклатурной группы.

Ведение информации о контрагентах предприятия.

Справочник "Контрагенты" предназначен для введения и хранения информации о юридических лицах и частных предпринимателях с которыми предприятие контактирует, осуществляя свою деятельность.

  • Поставщики, которые поствляют свой товар непосредственно в магазины;
  • Покупатели, осуществляющие закупки мелким оптом непосредственно в магазинах;
  • Банки, предоставляющие услуги эквайринга в магазинах торговой сети.

Зачем нужны виды контактной информации?

Справочник служит для классификации контактной информации в справочниках системы - например, контрагентов, сотрудников, покупателей и т.д.

В списке справочника виды контактной информации, относящиеся к одному справочнику объединены в группу. В системе поддерживается контактная информация для справочников:

  • Организации
  • Пользователи
  • Физические лица
  • Контрагенты

Для каждой группы определен список предопределенных видов контактной информации, которые отмечены в списке определенным знаком. При необходимости в любую из групп пользователь может добавить необходимый вид дополнительной контактной информации. После добавления новый вид контактной информации автоматически появится в форме соответствующего справочника.

Как создать запись о новом контрагенте?

В карточке Контрагента фиксируется необходимая для работы информация. Вводится наименование контрагента, как правило, соответствующее краткому наименованию его организации и указывается его правовой статус:

  • Юридическое лицо
  • Физическое лицо
  • Юридическое лицо за пределами Украины
  • Индивидуальный предприниматель

Для юридических лиц возможно указать полное наименование организации контрагента и ввести Код по ЕДРПОУ, ИНН и Номер свидетельства плательщика НДС, если юридическое лицо зарегиcтрировано на территории украины.

Для физических лиц указывается Фамилия, Имя и Отчество физического лица, Документ, удостоверяющий его личность и код ДРФО, если физическое лицо зарегистрировано в качестве Индивидуального предпринимателя.

В карточку вносится также Контактная информация и Дополнительная информация о контрагенте, в соответсвии с предварительными настройками ведения информации о контрагентах.

Ведение информации о физических лицах.

Справочник "Физические лица" содержит список физических лиц, работающих на предприятии (сотрудники и пользователи программы) или являющихся покупателями магазинов торговой сети. В справочнике можно объединять физических лиц в произвольные группы и подгруппы, выделяя, например, определенные категории покупателей.

Для физического лица определяются его контактные данные: адрес, телефон, е-майл и т.д. При необходимости пользователь может добавить для физического лица дополнительные виды контактной информации.

Данные о физических лицах, имеющих право оформлять документы и вести хозяйственную деятельность, вносится в справочник Контрагенты с указанием правового статуса Физическое лицо. В этом случае появляется возможность внести юридические данные физического лица, сведения о банковских счетах и поставляемых товарах.

Перед началом заполнения списка физических лиц необходимо выполнить настройки по ведению списка физических лиц:

Заполнить классификатор документов, чтобы при заполнении данных физического лица пользователь мог выбрать значение из готового списка.
Настроить виды контактной информации для ввода контактной информации по физическим лицам.
Указать список дополнительных свойств (дополнительных реквизитов и сведений).

Как используется справочник "Аналитика хозяйственных операций"?

Справочник предназначен для расширения аналитик учета таких хозяйственных операций, как Списание, Возврат от покупателя и Возврат поставщику.

В справочнике имеются предустановленные элементы для хозяйственных операций:

  • Поступление товаров
  • Реализация товаров
  • Перемещение товаров
  • Комплектация номенклатуры
  • Оприходование
  • Погашение подарочных сертификатов
  • Списание на затраты
  • Скидки подарки

Для создания нового аналитического признака необходимо открыть список аналитик, расположенной в меню Обслуживание информационной базы раздела Администрирование, и нажать в командной панели кнопку Создать. В открывшейся форме ввести наименование аналитики и указать из перечисления хозяйственную операцию, к которой она относится.

Как используется справочник "Статьи движения денежных средств"?

Справочник используется для детализации учета потока денежных средств. Чаще всего в более строгом контроле нуждается затратная часть расходов.

Пример.Для хозяйственной операции «Оплата поставщику» политикой предприятия в части учета денежных средств предусмотрены еще три статьи:
- Оплата поставщикам товара
- Оплата поставщикам за расходные материалы торгового процесса
- Оплата поставщикам за канцелярские товары
Таким образом, проанализировав отчет о движении денежных средств за период можно сделать вывод направлении расходования денежных средств.

Для создания нового аналитического признака необходимо открыть список Статей движения женежных средств, расположенной в меню Денежные средства раздела Финансы, и нажать в командной панели кнопку Создать. В открывшейся форме ввести наименование аналитики и указать из перечисления хозяйственную операцию, к которой она относится.

Настройка видов оплаты.

В программе поддерживается три вида безналичных оплат:

  • Оплата платежной картой банка (Эквайринг)
  • Оплата с помощью банковского кредита
  • Оплата подарочным сертификатом, выпущенным самим торговым предприятием.

Возможность использования оплат платежными картами управляется соответствующей настройкой параметров учета раздела Финансы (Администрирование). При этом становится доступно меню Эквайринг раздела Финансы, в котором необходимо внести данные о Договоре эквайринга с банком и эквайринговых терминалах, которые будут использоваться в магазинах.

Для использования оплаты с помощью банковского кредита также нужно установить флаг "Использовать оплаты банковским кредитом" в настройках параметров учета раздела Финансы. При этом в меню доступных видов оплат Рабочего места кассира становятся доступны Виды оплат с типом Банковский кредит.

Подарочные сертификаты - это фактически обязательства учесть сумму сертификата при оплате товара, которое берет на себя предприятие при продаже сертификата. Продажа и зачет оплаты производится по одной и той же цене - номиналу сертификата. Сертификат не именной, он может быть подарен или перепродан, что позволяет использовать сертификаты как аналог, заменитель денежных средств.

Где можно увидеть список Видов оплат?

Список оплат чеков ККМ можно открыть из меню Обслуживание информационной базы раздела Администрирование. В список выводится Наименование , Тип оплаты и Процент комиссии банка для типов оплат "Банковский кредит" и "Платежная карта".

Справочник содержит два предопределенных элемента:

  • Наличные - оплата наличными поддерживается программой при любых настройках;
  • Оплата подарочным сертификатом - становится доступной, если в программу введен хотя бы один подарочный сертификат.

Виды оплат с типами "Банковский кредит" и "Платежная карта" вводятся пользователем по мере необходимости. Создать новый вид оплаты можно с помощью кнопки Создать командного меню списка.

Как настроить оплаты платежными картами?

Чтобы настроить оплату чеков ККМ платежными картами нужно вести в программу информацию о поддерживаемых банком-эквайрером видах платежных карт. Эта информация вносится в Договор эквайринга.

Как просмотреть список договоров эквайринга?

Для того чтобы в магазине была возможность регистрации оплаты платежными картами, необходимо предварительно оформить Договоры эквайринга с банком для обслуживания платежных карт.

Список договоров эквайринга можно вызвать из меню Эквайринг раздела Финансы. Для удобства пользователя список может быть установлен быстрый отбор по Организации. Для создания нового договора нужно нажать кнопку Создать в командном меню списка.

Какая информация вводится в договор эквайринга?

Договор эквайринга предназначен для регистрации видов платежных карт и условий оплаты услуг банка-эквайрера. Порядок оформления договора эквайринга следующий:

В справочник Контрагенты (меню Контрагенты раздела Нормативно-справочной информации) ввести информацию о банке, который будет обслуживать платежные карты.

В табличной части нужно выбрать из справочника Виды оплат виды платежных карт, принимаемых к оплате по данному договору. При необходимости можно создать новый вид платежной карты, нажав в списке кнопку Создать. Созданный таким образом элемент будет иметь заполненный тип оплаты Платежная карта, останется ввести лишь наименование.

Для каждого вида платежной карты можно установить свой процент банковской комиссии за проведение платежа. Если по условиям договора банк-эквайрер возвращает комиссию при отмене платежа в течение банковского дня, то следует установить флажок Эквайрер возвращает комиссию при возврате. Возврат комиссии в течение банковского дня будет учтен при расчете суммы выплат по эквайрингу за период.

В Договоре эквайринга также перечисляются эквайринговые терминалы, с помощью которых производится оплата платежной картой. Список терминалов можно открыть из меню Эквайринговые терминалы панели навигации формы договора.

Как зарегистрировать в системе эквайринговые терминалы?

С помощью платежной карты можно оплатить как розничную так и мелкооптововую покупку. Поэтому при создании новой записи об эквайринговом терминале необходимо указать Тип кассы – операционная касса или касса ККМ - и выбрать из соответствующего списка касс данной организации нужную кассу. Наименование терминала формируется в автоматическом режиме по наименованию банка-эквайрера и организации. Пользователь может изменить наименование, а также ввести Идентификационный код терминала.

Если платежи будут проводиться без подключения терминала к компьютеру пользователя, то следует установить флажок Использовать без подключения оборудования. В противном случае для каждого терминала следует предварительно создать запись в справочнике Подключаемое оборудование.
