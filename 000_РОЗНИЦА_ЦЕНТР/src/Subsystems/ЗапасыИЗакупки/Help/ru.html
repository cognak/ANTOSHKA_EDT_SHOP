Как просмотреть список заказов на перемещение?

Список документов можно открыть из пункта Заказы на перемещение меню Складские операции раздела Заказы и закупки.

Для удобства работы Пользователь может установить отбор по полям Магазин-заказчик и Магазин-поставщик. По каждому заказу в списке выводится желаемая дата поставки, менеджер, оформивший заказ и организация на которую нужно переместить товар.

На основании выбранного в списке документа можно создать документ "Перемещение товаров" и распечатать Заказ на перемещение.

Как сформировать заказ на распределительный склад?

В документе указывается магазин - заказчик, на который требуется переместить товар, магазин - поставщик или распределительный центр, желаемая дата поступления товаров и список товаров, которые необходимо переместить.

Строки документа можно ввести вручную выбором из справочника «Номенклатура» или с помощью подбора. Можно также считать штриховой код товара с помощью сканера, если оформляется заказ на пополнение недостаточного количества товара имеющегося на остатке в магазине.

Заказы на перемещение могут также формироваться с помощью обработки "Планирование потребности".

Какие возможности подбора товара предоставляет система?

Пользователь может Установить вариант подбора: По иерархии номенклатуры, По виду номенклатуры и По сегментам номенклатуры. В левом окне подбора отображаются элементы выбранного пользователем вида подбора. В правом окне формы подбора отображается номенклатура, входящая в выбранный элемент по виду отбора: в сегмент номенклатуры, в выбранную группу номенклатуры или номенклатура выбранного вида.

Список характеристик выбранной в правом окне номенклатурной позиции можно открыть двойным щелчком по позиции или нажав кнопку Характеристики. С помощью кнопки Остатки можно просмотреть в отдельном окне текущие остатки по местам хранения выбранной номенклатуры. Остатки можно отображать либо По магазину документа или По всем магазинам.

Для быстрого поиска определенной позиции номенклатуры можно использовать строку поиска по наименованию номенклатуры. В правом окне выведется список номенклатуры, содержащий введенную в поле поиска строку и принадлежащий выбранному элементу вида отбора. При установке флага «Использовать полнотекстовый поиск при подборе» в разделе Администрирование область поиска расширяется на дополнительные реквизиты и свойства номенклатуры, характеристики номенклатуры и штрихкоды.

Подбор товара в документ производится в два этапа: сначала товар размещается в список отобранных товаров подбора и затем переносится в документ с помощью кнопки Перенести в документ. Товар помещается в список двойным щелчком по нужной строке. В процессе добавления товара система предоставляет возможность вводить количество и цену, если пользователь нажмет на кнопку Запрашивать количество и цену.

Как найти и добавить элемент справочника в табличную часть по штрихкоду?

Чтобы добавить элемент справочника по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Для ручного ввода символов штрихового кода нужно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную.

Если данный штрихкод зарегистрирован в системе, соответствующий ему элемент будет добавлен в табличную часть. В противном случае в правой части формы появится ссобщение об ошибке. Если элемент с данным штрихкодом уже введен в табличную часть, то строка с искомым элементом будет выделена.

Как планировать поставки с помощью сервиса "Планирование потребности в номенклатуре"?

В настройках обработки указываются следующие показатели:

  • Период анализа - период, на основании которого рассчитываются средние продажи.
  • Период планирования - период, на который планируется проводить закупки.
  • Магазин – магазин, по которому планируется потребности в номенклатуре. Порядок округления количественных характеристик отчета.

При планировании можно также указать, использовать или нет при расчете необходимого объема закупок те товары, которые планируются к поступлению по ордерной схеме.

Важно. Расчет дней продажи производится с учетом того графика работы, который установлен для магазина. Пред началом работы с отчетом необходимо для магазинов ввести информацию о графиках работы магазина. Для этого в карточке магазина надо нажать на кнопку График работы, выбрать график работы, применяемый в магазине и нажать на кнопку ОК.

В качестве показателей отчета выводятся следующие показатели:

  • Количество дней продажи. Если остаток товара на конец анализируемого периода продаж больше нуля, то - рассчитывается количество дней, которое товар находится в продаже, с момента последнего «нулевого» остатка на складе магазина до конца анализируемого периода продаж. Если в течение анализируемого периода остаток ни разу не обнулялся, то количество дней считается с начала анализируемого периода продаж до конца анализируемого периода продаж. Если остаток товара на конец анализируемого периода продаж меньше или равен нулю, то – рассчитывается количество дней с предыдущего нулевого остатка до момента нынешнего обнуления. Если предыдущий нулевой остаток отсутствует в анализируемом периоде, то в качестве даты предыдущего обнуления используется начальная дата анализируемого периода продаж.
  • Количество продаж. Общее количество проданного товара за анализируемый период продаж.
  • Количество возвратов. Общее количество возвращенного товара.
  • Среднее количество продаж. Рассчитывается как отношение проданного количества за период между последними двумя «обнулениями» остатков, к количеству рассчитанных дней продажи. В случае отсутствия одной/обеих дат обнуления остатков, используется дата начала или/и конца анализируемого периода продаж соответственно.
  • Среднее количество возвратов. Рассчитывается как отношение возвращенного количества за период между последними двумя «обнулениями» остатков, к количеству рассчитанных дней продажи. В случае отсутствия одной/обеих дат обнуления остатков, используется дата начала или/и конца анализируемого периода продаж соответственно.
  • Количество дней планирования. Количество планируемых дней продажи рассчитывается в соответствии с указанным периодом планирования и графиком работы магазина.
  • Планируемое количество продаж. Рассчитывается, как произведение показателей: количество дней планирование и среднее количество продаж.
  • Планируемое количество возвратов. Рассчитывается, как произведение показателей: количество дней планирование и среднее количество возвратов.
  • Остаток на начало планирования. Остаток товара на складах магазина на конец анализируемого периода продаж.
  • Количество к заказу или возврату. Количество товара, которое необходимо заказать или вернуть, если товар не продается. Рассчитывается по следующей формуле: Планируемое количество продаж - Остаток на начало планирования - Планируемое количество возвратов

Положительное значение показателя означает, что товар следует закупить для магазина, отрицательное количество товара означает, что товар можно вернуть, так как он не будет продан в планируемом периоде.

Пример.В соответствии с данными в отчете планируется продажа 100 штук товаров, планируемое количество возвращаемого товара 10 штук, остаток на складе 150 штук. Количество к заказу или возврату (100-150-10 = -60). Это означает, что по данному товару с учетом процента возвращаемого товара есть излишки и 60 штук товара можно вернуть или принять меры к повышению количества продаж данного товара, может быть переместить его в другой магазин.

Для того, чтобы в отчете оставить только те товары, которые планируется закупать, следует в настройках параметров отбора установить отбор со следующими параметрами: Количество к заказу или возврату больше 0

На основании этого документа может быть оформлен документ "Перемещение товаров". Документ перемещения может быть оформлен между складом центрального офиса и магазином или между различными магазинами, входящими в состав центрального узла.

Какие печатные формы предусмотрены для документа "Заказ на перемещение"?

Для документа предусмотрены печатная форма Заказ на перемещение. которую можно использовать, например для передачи заказа по факсу.

Перемещение товаров.

Поставки товара с распределительного центра оформляются документом "Перемещение товаров". При этом документы оформляются в управляющей системе ("1С:Управление торговлей для Украины" , ред.3) и мигрируют в конфигурацию "1С: Розница для Украины".

Кроме того, в процессе реализации товаров периодически может возникать потребность в перераспределении товаров между магазинами. Например, в случае неравномерных или несоответствующих прогнозу продаж. В этом случае Менеджер центрального офиса или магазина на основании проведенного анализа продаж и остатков оформляет документы перемещения товаров в другой магазин.

Потребность переместит товары из магазина в магазин может возникнуть и тогда, когда его достаточно дорогого товара не хватает для экспозиции в каждом магазине сети. В этом случае продажа производится преимущественно по каталогам, а отобранный покупателем товар перемещается в магазин для просмотра согласно документам "Заказ на перемещение".

И при работе с управляющей системой и при перемещении между магазинами, оформление документов "Перемещение товаров" всегда производится по ордерной схеме. Для ускорения оформлений отгрузки и поступления используется преднастроенные схемы автоматического распределения товара по складам магазина.

Как просмотреть список документов перемещения?

Список документов можно открыть из пункта Перемещения товаров меню Закупки раздела Заказы и закупки.

Для выбранного документа в командной панели списка доступны сервисы:

  • Распределить отгрузку по складам - обработка распределения отгрузки по складам хранения товара магазина.
  • Распределить поступление по складам - обработка распределения поступления по складам (отделам) магазина.
  • Создать на основании - создание документов на основании выбранного элемента списка.
  • Печать документа - распечатка доступных для данного типа документа форм.

Для удобства работы Пользователь может установить отбор по полям Магазин-отправитель, Склад-отправитель, Магазин-получатель и Склад-получатель. По каждому документу в списке выводится документ-основание, дата и сумма документа, а также расхождения, возникшие при поступлении.

Как оформить документ "Перемещение товаров"?

Для ввода нового документа перемещения товара из магазина в магазин нужно нажать кнопку Создать в списке документов Перемещения товаров и в открывшейся форме заполнить поля шапки документа: магазин и склад отправителя и магазин и склад получателя. По умолчанию поля отправителя заполняются в соответствии с настройками Пользователя, а если они не установлены, то значениями быстрого отбора. Если справочник содержит единственный элемент, то он будет подставлен по умолчанию.

Чтобы добавить в документ номенклатуру нужно нажать кнопку Добавить и выбрать из нужную позицию из справочника Номенклатуры. Помимо ручного подбора можно воспользоваться или воспользоваться сервисом Поиск по штриховому коду или Подобрать товары в командной панели табличной части.

Для каждой товарной строки указывается:

  • Характеристика номенклатуры – можно выбрать из списка характеристик данной номенклатуры.
  • Упаковка – по умолчанию указывается основная единица изменрения, но может быть выбрана любая упаковка данной номенклатуры
  • Количество – количество поступивших единиц или упаковок
  • Цена – заполняется специальным сервисом
  • Ставка НДС – заполняется по указанной для данной номенклатуры ставке НДС.
  • Сумма НДС вычисляется по ставке НДС данной номенклатуры
  • Сумма по строке рассчитывается на основании введенных данных.

Как заполнить цены товаров?

Цены в строках документа можно ввести вручную или воспользоваться сервисами заполнения цены:

  • Заполнить цены по розничным ценам – цены перемещения будут установлены по розничным ценам магазина отправителя.
  • Заполнить цены выделенных строк по розничным ценам – будут заполнены цены только выделенных пользователем строк.
  • Заполнить цены по виду цен – цены перемещения будут установлены по выбранному пользователем виду цен, например специальные трансферные цены для перемещения между магазинами.
  • Заполнить цены выделенных строк по виду цен – по выбранному виду цен будут заполнены цены только выделенных пользователем строк.

Как создать документ "Перемещение товара" на основании документа "Заказ на перемещение"?

При этом реквизиты шапки документа и товарные строки заполняются из указанного документа и вносится информационная запись о документе-основании.

На основании документа "Перемещение товаров" можно оформить следующие документы:

  • Приходный ордер - документ, увеличивающий складские остатки.
  • Расходный ордер - документ, уменьшающий складские остатки.

Как распределить товары по складам магазина?

Нажмите кнопку Распределить отгрузку по складам или Распределить поступление по складам в командном меню списка документов или в самом документе. В открывшейся форме список товаров, количество к распределению и склад приходования заполняется по документу. Эту информацию можно изменить.

Информация о складах может быть заполнена вручную для каждой строки из справочника Склад или автоматически. Для автоматического заполнения нажмите кнопку Заполнить склад и выберите вариfнт заполнения данных:

  • По правилам распределения - товары должны поступить в торговые залы в соответствии с указанной схемой распределения по номенклатурным группам.
  • Из справочника – в случае, если весь товар должен поступить на склад, отличный от указанного в документе нужно выбрать нужный из справочника складов.
  • Из документа - используется для восстановления значения склада из документа.

Те же способы заполнения складов доступны только для выбранных строк.

Фактическое количество приходуемого на указанные склады товара также можно изменить вручную или групповым заполнением. Доступны варианты заполнения количества:

  • По документу - для восстановления количества указанного в документе «Поступление номенклатуры».
  • Фиксированным значением – для всех строк документа устанавливается введенное пользователем значение. Это может быть полезно, например, при отборе образцов поступившего товара для витрины.

Для распределяемых подарочных сертификатов можно указать серийные номера из числа перечисленных в документе, нажав на кнопку Ввести подарочные сертификаты.

По окончании ввода информации пользователь нажимает кнопку Сформировать ордера. При этом будут сформированы Расходные ордера или Приходные ордера на указанные в таблице позиции номенклатуры с фактическим количеством на указанные склады.
Если одна и та же позиция должна быть размещена на несколько складов обработку нужно повторить. При повторном вызове сервиса табличная часть заполнится нераспределенными позициями номенклатуры с оставшимся количеством.

Анализ потребности в товарах.

Основная задача Менеджера по закупкам – определить поставщиков и товарные позиции с недостаточным текущим товарным остатком для оформления документов «Заказ поставщику» или «Заказ на перемещение» в магазин с распределительного центра сети.

Определить потребность в товарах можно как с помощью Отчетов о продажах, так и с помощью обработки "Анализ продаж и формирование Заказов поставщику".

Как в том, так и в другом случае основным требованием является совпадение учетных данных о текущих товарных остатков и фактических остатков товара на полках магазина. Это достигается с помощью регулярных инвентаризаций в отделах торгового зала и подсобных складских помещениях.

В Рознице поддерживается упрощенный подход к поставкам заказанной номенклатуры. Предполагается, что по каждому документу "Заказ поставщику" будет зарегистрировано только одно "Поступление товаров" в оговоренный с поставщиком период. Такой подход позволяет в автоматическом или полуавтоматическом режиме закрывать просроченные заказы, что помогает Менеджеру сосредоточится на достижении своей цели – обеспечить наличие товара на полках магазина.

Как создать Заказ поставщику с помощью сервиса "Анализ продаж и формирование заказов поставщикам"?

Вызовите обработку "Анализ продаж и формирование Заказов поставщику" из меню "Сервис" раздела "Запасы и закупки" и введите параметры расчета:

  • Магазин - по умолчанию устанавливается текущий магазин, если он входит в число доступных пользователю;
  • Дата заказа - устанавливается текущая дата;
  • Дата поставки - планируемая дата поставки;
  • Поставщик - указывать поставщика не обязательно, в этом случае в результат выведутся все поставщики, которые поставляют указанный в отборе товар;
  • Количество дней торговли и исходящий остаток, которые должен обеспечить заказ;
  • Период анализа продаж - характерный для выбранной категории статистический период, за который рассчитываются средние продажи, определяется экспертным путем.

Пользователь может использовать отбор по определенной группе товаров, воспользовавшись сервисом Настройка.

Пример.В пятницу 14 августа менеджеру нужно оформить заказ на молочную продукцию с поставкой на утро субботы 15 августа на 2 дня торговли: субботу и воскресенье, 15 и 16 августа. Причем на утро понедельника исходящего остатка должно хватить примерно на 1 день торговли. Он знает, что в выходные дни потребление несколько выше, чем в будни. Поэтому для анализа статистики продаж выбирает последние выходные дни (8 и 9 августа), указывает количество дней торговли (2 дня), на которые система рассчитает заказ, планируемый исходящий остаток на 1 день и предполагаемую дату поставки – 15 августа, ограничивает номенклатуру группой «Молочная продукция».

Как производится расчет потребности с помощью сервиса "Анализ продаж и формирование заказа поставщику"?

Система рассчитывает рекомендованное к заказу количество по формуле: Количество к заказу = (Количество продаж за период анализа / Количество дней в периоде анализа) * (Дата планируемой поставки - Дата заказа + Количество дней в периоде планирования + Исходящий остаток в днях торговли) - (Текущий остаток + Количество заказано).

При расчете потребности в товарах учитываются количество товара, уже размещенное в Заказах. Если не используется регламентное задание автоматического закрытия просроченных заказов, то перед расчетом потребности рекомендуется воспользоваться сервисом "Закрытие невыполненных Заказов поставщику".

Расчет потребности не учитывает занижение среднедневных продаж из-за отсутствия товара на полке в период статистики и колебания спроса. Таким образом, лучше всего алгоритм работает для товаров хорошо обеспеченного стабильного спроса. В случае редких или нестабильных продаж резкий тренд роста или падения, закупки нового товара, количество товара к заказу определяются экспертным путем.

Так как алгоритм не учитывает тенденции спроса, точность расчета рекомендованного к заказу количества падает, если период поставки (от даты заказа до даты поступления) слишком большой.

Важно. Вывод информации о потребности в группировке по поставщику имеет особенность: так как поставщиков одной и той же номенклатуры может быть несколько, необходимо учитывать, что позиция номенклатуры будет задублирована для каждого поставщика.
Например, поставщики "Первый" и "Второй" поставляют одну и ту же номенклатуру «Ботинки». Рассчитанная потребность «Ботинки» – 10 пар. В результате обработка рекомендует к заказу: поставщик "Первый": «Ботинки» – 10 пар; поставщик "Второй": «Ботинки» – 10 пар.
Эту особенность необходимо учитывать, формируя заказы поставщикам последовательно, обновляя данные в обработке после каждого заказа.

Какая информация выводится менеджеру для принятия решения?

По умолчанию товарные строки будут сгруппированы по поставщикам и выведены в порядке убывания потребности. Сами поставщики будут выведены в порядке уменьшения суммы заказа. Это позволит менеджеру сконцентрироваться на поставщиках с наибольшим заказом и на товарах с наибольшей потребностью.

Товарные строки выводятся с количественными и суммовыми показателями, сгруппированными в четыре блока: Продажи, Остатки, Заказано и К заказу.
Информацию по заказанным товарам можно уточнить с помощью расшифровки отчета по двойному клику на ячейке Заказано кол-во. При этом раскрывается список заказов, в котором есть данный товар.

В качестве дополнительной информации выводится дата последнего поступления товара, цена последней поставки и розничная цена товара. Этой информации обычно достаточно для принятия решения.

Выбрав поставщика в параметрах обработки, менеджер отмечает те позиции номенклатуры, которые необходимо заказать, корректирует по необходимости количество к заказу и формирует документ Заказ поставщику с помощью команды Сформировать заказ. При этом отмеченные товары будут перенесены в документ с указанным пользователем количеством.

Ведение справочника "Поставщики".

Информация о тех поставщиках, которые осуществляют поставку товаров непосредственно в магазин вносится в справочник Контрагенты.

Справочник иерархический, поставщики могут быть сгруппированы произвольным удобным для пользователя образом. Например, поставщики определенной номенклатурной группы или поставщики с определенными условиями поставки.

В системе также регистрируется информация о номенклатуре, поставляемой данным поставщиком в определенный магазин. Информацию о номенклатуре поставщика можно просмотреть, перейдя в список в меню карточки поставщика.

Сумма заказа рассчитывается в соответсвии с настройками учета НДС на закладке Дополнительно. При установленных флагах Учитывать НДС и Цена включает НДС сумма документа будет содержать рассчитанную по указанным в карточке номенклатуры ставкам сумму НДС.

Закрытие заказа.

Информация о заказанной номенклатуре используется при анализе продаж и определении потребности в товарах. Актуальность данной информации поддерживается с помощью закрытия заказов, которое может производится в двух случаях: при поступлении товара или отмене заказа поставщику.

Поддерживаемая системой технология не подразумевает частичные поставки по заказу. Это позволяет планировать нагрузку отдела приемки магазина и учитывать уровень сервиса поставщика при исполнении заказов. Если предполагается дополнительная поставка товара, то рекомендуется закрыть частично исполненный заказ и его основе создать новый заказ, в котором указать недопоставленные позиции и новую дату поставки.

Отмену заказа поставщику рекомендуется производить в случае, если поставщик не смог привезти товар в указанный срок. В этом случае рекомендуется пересмотреть потребность в товаре и заново сформировать заказ с учетом отмененной поставки.

Нажмите кнопку Распределить отгрузку по складам или Распределить поступление по складам в командном меню списка документов или в самом документе. В открывшейся форме список товаров, количество к распределению и склад приходования заполняется по документу. Эту информацию можно изменить.

Информация о складах может быть заполнена вручную для каждой строки из справочника Склад или автоматически. Для автоматического заполнения нажмите кнопку Заполнить склад и выберите вариfнт заполнения данных:

  • По правилам распределения - товары должны поступить в торговые залы в соответствии с указанной схемой распределения по номенклатурным группам.
  • Из справочника – в случае, если весь товар должен поступить на склад, отличный от указанного в документе нужно выбрать нужный из справочника складов.
  • Из документа - используется для восстановления значения склада из документа.

Те же способы заполнения складов доступны только для выбранных строк.

Фактическое количество приходуемого на указанные склады товара также можно изменить вручную или групповым заполнением. Доступны варианты заполнения количества:

  • По документу - для восстановления количества указанного в документе «Поступление номенклатуры».
  • Фиксированным значением – для всех строк документа устанавливается введенное пользователем значение. Это может быть полезно, например, при отборе образцов поступившего товара для витрины.

Для распределяемых подарочных сертификатов можно указать серийные номера из числа перечисленных в документе, нажав на кнопку Ввести подарочные сертификаты.

По окончании ввода информации пользователь нажимает кнопку Сформировать ордера. При этом будут сформированы Расходные ордера или Приходные ордера на указанные в таблице позиции номенклатуры с фактическим количеством на указанные склады.
Если одна и та же позиция должна быть размещена на несколько складов обработку нужно повторить. При повторном вызове сервиса табличная часть заполнится нераспределенными позициями номенклатуры с оставшимся количеством.

Товар помещается в список двойным щелчком по нужной строке. В процессе добавления товара система предоставляет возможность вводить количество и цену, если пользователь нажмет на кнопку Запрашивать количество и цену.

Если данный штрихкод зарегистрирован в системе, соответствующий ему элемент будет добавлен в табличную часть. В противном случае в правой части формы появится ссобщение об ошибке. Если элемент с данным штрихкодом уже введен в табличную часть, то строка с искомым элементом будет выделена.

Как оформить отгрузку по ордерной схеме возврата поставщику?

Возврат товаров поставщику может оформляться по ордерной схеме. Такая схема применяется в том случае, если документ возврата оформляет менеджер, а факт списания товара со склада фиксируется кладовщиком. При этом предусматривается следующий порядок оформления документов:

  • Менеджер оформляет документ "Возврат товаров поставщику".
  • На основании проведенного документа Возврат товаров поставщику оформляется документ "Расходный ордер на товары".
  • Кладовщик проводит документ "Расходный ордер на товары" и фиксирует факт отгрузки (возврата) товаров поставщику.

Настройка ордерной схемы производится для каждого магазина отдельно. Если ордерная схема в данном магазине не используется, то факт отгрузки товара поставщику фиксируется в момент проведения документа Возврат товаров поставщику.

Как можно проанализировать причины возвратов поставщику?

Отчет «Причины возвратов поставщику» предназначен для суммовой оценки возвратов постащику в разрезе магазинов и причин возвратов, указанных пользователем в справочнике Аналитики хозяйственных операций.
Показатели отчета:

  • "Средняя себестоимость единицы товара, грн. - определяется среднее значение себестоимости за период отчета
  • Средняя розничная цена за период отчета, грн. - определяется среднее значение розничной цены за период отчета
  • Количество возврата - количество возвращенной номенклатуры
  • Сумма возврата, грн - сумма возврата в ценах закупки
  • Количество поступления - количество поступившей номенклатуры за период отчета
  • Сумма поступления, грн. - сумма поступления в ценах закупки
  • Доля возврата в поступлении, % - отношение суммы возврата к сумме поступления в процентах."

Отчет формируется только по возвращенной поставщику за отчетный период номенклатуре.

Как оформить поступление денежных средств от поставщика при возврате товаров?

В конфигурации «Розница для Украины» не ведутся взаиморасчеты с поставщиками. Однако поставщик может вернуть деньги за возвращенные ему товары непосредственно в операционную кассу магазина.

Такая операция оформляется документом "Приходный кассовый ордер" с установленным видом операции Возврат денежных средств поставщиком.

Документ может быть оформлен на основании документа "Возврат товаров поставщику". В этом случае в документе РКО указывается документ-основание и сумма расходного кассового ордера автоматически заполняется суммой возвращаемых товаров, указанных в документе возврата.

Если же товар поставляется непосредственно в магазин, то необходимо некоторое время для того, чтобы подготовить его к продаже – распечатать ценники и этикетки в удобное для этого время, например до открытия магазина. Как оформить приходный ордер на товары?

Для создания ордера необходимо найти в списке соответствующих документов нужный документ и выбрать пункт Приходный ордер в списке меню Создать на основании. Товары, отсутствующие в документе-основании при необходимости можно добавить с помощью кнопки Добавить командного меню, с помощью сканера штрихкода или сервиса Подобрать товары. Пользолватель может также детализировать информацию о сертификатах, введя необходимые серийные номера.

Какие возможности подбора товара предоставляет система?

Пользователь может Установить вариант подбора: По иерархии номенклатуры, По виду номенклатуры и По сегментам номенклатуры. В левом окне подбора отображаются элементы выбранного пользователем вида подбора. В правом окне формы подбора отображается номенклатура, входящая в выбранный элемент по виду отбора: в сегмент номенклатуры, в выбранную группу номенклатуры или номенклатура выбранного вида.

Список характеристик выбранной в правом окне номенклатурной позиции можно открыть двойным щелчком по позиции или нажав кнопку Характеристики. С помощью кнопки Остатки можно просмотреть в отдельном окне текущие остатки по местам хранения выбранной номенклатуры. Остатки можно отображать либо По магазину документа или По всем магазинам.

Для быстрого поиска определенной позиции номенклатуры можно использовать строку поиска по наименованию номенклатуры. В правом окне выведется список номенклатуры, содержащий введенную в поле поиска строку и принадлежащий выбранному элементу вида отбора. При установке флага «Использовать полнотекстовый поиск при подборе» в разделе Администрирование область поиска расширяется на дополнительные реквизиты и свойства номенклатуры, характеристики номенклатуры и штрихкоды.

Подбор товара в документ производится в два этапа: сначала товар размещается в список отобранных товаров подбора и затем переносится в документ с помощью кнопки Перенести в документ. Товар помещается в список двойным щелчком по нужной строке. В процессе добавления товара система предоставляет возможность вводить количество и цену, если пользователь нажмет на кнопку Запрашивать количество и цену.

Как найти и добавить элемент справочника в табличную часть по штрихкоду?

Чтобы добавить элемент справочника по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Для ручного ввода символов штрихового кода нужно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную.

Если данный штрихкод зарегистрирован в системе, соответствующий ему элемент будет добавлен в табличную часть. В противном случае в правой части формы появится ссобщение об ошибке. Если элемент с данным штрихкодом уже введен в табличную часть, то строка с искомым элементом будет выделена.

Как добавить серийный номер сертификата в документ?

Если сертификаты имеют серийные номера, в документе можно указать диапазон серийных номеров воспользовавшись командой Ввести подарочные сертификаты в меню табличной части документа. В открывшейся форме доступен выбор Диапазона из числа существующих или предоставляется возможность создать новый диапазон серийных номеров подарочных сертификатов.

  • Реализация товаров мелким оптом
  • Возврат товара поставщикам
  • Перемещение товаров между магазинами

Для создания ордеров по документам "Возврат товаров поставщику" и "Реализация товаров" нужно установить настройку использования ордерной схемы в данном магазине. Перемещение товаров из магазина в магазин всегда производится по ордерной схеме.

Распределение товара по складам магазина.

Номенклатура, поступившая в магазин по умолчанию оприходуется на склад поступления, указанный в настройках магазина.

При использовании ордерной схемы для поступления имеется возможность настроить автоматическое распределение номенклатуры на различные склады магазина (отделы) для разных номенклатурных групп. Настройки распределения задаются для каждого магазина отдельно.

Использование автоматического распределения позволяет быстро и безошибочно оприходовать поступивший товар в торговые залы магазина.

Как распределить товары по складам магазина?

Как распределить товары по складам магазина?

Нажмите кнопку Распределить отгрузку по складам или Распределить поступление по складам в командном меню списка документов или в самом документе. В открывшейся форме список товаров, количество к распределению и склад приходования заполняется по документу. Эту информацию можно изменить.

Информация о складах может быть заполнена вручную для каждой строки из справочника Склад или автоматически. Для автоматического заполнения нажмите кнопку Заполнить склад и выберите вариfнт заполнения данных:

  • По правилам распределения - товары должны поступить в торговые залы в соответствии с указанной схемой распределения по номенклатурным группам.
  • Из справочника – в случае, если весь товар должен поступить на склад, отличный от указанного в документе нужно выбрать нужный из справочника складов.
  • Из документа - используется для восстановления значения склада из документа.

Те же способы заполнения складов доступны только для выбранных строк.

Фактическое количество приходуемого на указанные склады товара также можно изменить вручную или групповым заполнением. Доступны варианты заполнения количества:

  • По документу - для восстановления количества указанного в документе «Поступление номенклатуры».
  • Фиксированным значением – для всех строк документа устанавливается введенное пользователем значение. Это может быть полезно, например, при отборе образцов поступившего товара для витрины.

Для распределяемых подарочных сертификатов можно указать серийные номера из числа перечисленных в документе, нажав на кнопку Ввести подарочные сертификаты.

По окончании ввода информации пользователь нажимает кнопку Сформировать ордера. При этом будут сформированы Расходные ордера или Приходные ордера на указанные в таблице позиции номенклатуры с фактическим количеством на указанные склады.
Если одна и та же позиция должна быть размещена на несколько складов обработку нужно повторить. При повторном вызове сервиса табличная часть заполнится нераспределенными позициями номенклатуры с оставшимся количеством.

Оофрмление перемещений между складами магазина.

Перемещение товаров между складами одного магазина производится с помощью документа Ордер на перемещение. Как правило, это перемещения товаров в торговый зал из подсобных помещений или перемещение выявленного при продаже брака в подсобные помещения магазина. Никаких настроек для использования ордеров на перемещение не требуется.

Ордер на перемещение может быть создан на основании документа "Поступление товаров". При этом товарные строки и склад-отправитель заполняется по документу основанию.

Как создать ордер на перемещение с помощью обработки "Анализ остатков"?

Обработка предназначена для определения количества товара, который нужно передать в торговый зал со складов магазина. Обработка анализирует средние продажи за указанный пользователем период, сравнивает их с остатками товара в торговом зале и рассчитывает количество к перемещению с учетом указанного менеджером желаемого периода торговли и исходящего остатка.

Пример.В пятницу утром кладовщик продовольственного магазина собирается переместить товар со склада в торговый зал. Он вводит период анализа продаж пятница и суббота прошлой недели, период торговли – 1 день, исходящий остаток – 0.5 дня. Обработка рассчитает потребность = средние продажи за период * количество дней торговли – (текущий остаток в торговом зале - исходящий остаток). Система выведет на форму товары, которые нужно подать в зал и они имеются в наличии на складе.

Пользователь заполняет параметры обработки:

  • Склад-получатель - Отдел в торговом зале магазина.
  • Склады-отправители - Кладовщик может указать, на какой торговый зал (отдел) и с каких складов будет осуществляться поставка. В этом случае сформируется список товара только с указанных складов по потребности указанного отдела. Отмеченные красным ячейки содержат информацию о товаре, которого не хватает на складе для подачи в зал (дефицит). Если товар к перемещению хранится на нескольких складах, такие строки отмечаются восклицательным знаком.
  • Период анализа - период, за который система агрегирует и рассчитывает средние продажи;
  • Дата заказа - текущая дата.

Пользователь может настроить дополнительно отборы и группировки с помощью команды Перейти к отбору.

Система рассчитывает рекомендуемое количество товара К перемещению, исходя из указанного пользователем количества дней торговли и количество дней продаж планируемого исходящего остатка. Выводятся только те товары, у которых текущий остаток меньше указанного в днях торговли.
Номенклатура группируется по складу-отправителю. По умолчанию сортируются по возрастанию показателя Остаток в днях торговли.

В сформированном списке кладовщик отмечает строки (по умолчанию отмечены все), по которым нужно сформировать документ перемещение. Документ формируется при нажатии кнопки Сформировать ордера в левом верхнем углу формы. Сформируется документ "Ордер на перемещение" по указанным строкам. Пользователь может записать и провести документ. Для формирования следующего документа нужно перерассчитать потребность.

Важно. В колонке "К перемещению" отображается потребность в товаре. Если он находится на нескольких указанных менеджером складах, то будет выведено несколько строк с остатками в разрезе складов, но с одинаковой потребностью. Рекомендуется формировать документы "Ордер на перемещение" последовательно с каждого склада, пересчитывая потребность после каждого документа.
